The most common performance optimization in frontend engineering is the **Cache-Aside Pattern**. 

Instead of blindly requesting data from an API every time a component loads, your application should first check its own `localStorage`.



### The Logic Gate
Because `localStorage.getItem("key")` returns `null` if the data doesn't exist, we can use a simple `if` statement to route our logic.

```javascript
async function fetchWithCache() {
  // 1. Check the local memory bank
  const cachedData = localStorage.getItem("user_profile");

  if (cachedData) {
    // 2. Cache HIT: We have the data! Parse it and use it instantly.
    console.log("Loaded from cache:", JSON.parse(cachedData));
    return; // Exit the function early so we don't hit the API
  }

  // 3. Cache MISS: The data isn't here. We must fetch it.
  console.log("Cache miss. Fetching from network...");
  const res = await fetch("[https://api.example.com/profile](https://api.example.com/profile)");
  const liveData = await res.json();

  // 4. Save the new data to the cache for next time!
  localStorage.setItem("user_profile", JSON.stringify(liveData));
  
  console.log("Loaded from network:", liveData);
}
```
