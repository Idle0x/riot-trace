Not all requests succeed. Sometimes the database is down. Sometimes the user is unauthorized. Sometimes the URL has a typo.

When a server replies, it attaches a 3-digit **Status Code** to the response header.



### The Standard Codes
* **200s (Success):** Everything went perfectly. (e.g., `200 OK`, `201 Created`).
* **400s (Client Error):** You messed up. (e.g., `401 Unauthorized`, `404 Not Found`).
* **500s (Server Error):** The backend messed up. (e.g., `500 Internal Server Error`).

### Handling Failures Safely
`fetch()` has a strange quirk: **It does not throw an error for 400 or 500 status codes.** It considers the request "successful" because the server successfully replied with an error! 

If you try to run `.json()` on a 404 page, your app will crash because it's trying to parse an HTML error page as JSON.

To protect your app, you must check the `res.ok` boolean. It is only `true` if the status is in the 200s.

```javascript
const res = await fetch("[https://api.example.com/data](https://api.example.com/data)");

if (res.ok) {
  // It is a 200-level success. Safe to parse.
  const data = await res.json();
  console.log(data);
} else {
  // It is a 400 or 500-level error.
  console.log("Request failed with status:", res.status);
}
```
