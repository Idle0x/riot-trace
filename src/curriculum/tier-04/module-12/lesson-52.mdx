### EXECUTION REQUIREMENTS
This is a Boss Fight. The terminal is yours. You are building a complete, production-grade network request.

1. Write an `async` function named `registerUser`.
2. Inside it, use `fetch()` to send a **POST** request to the provided `apiUrl`.
3. Include the `newUser` object as the body (properly stringified).
4. Include the proper `Content-Type` header.
5. Once the request completes, check if the response was successful using `res.ok`.
6. If successful, parse the JSON and log the new user's `id` (the API will assign one).
7. If it fails, log `"Registration Failed"`.
8. Execute your function.

If you skip the headers, the server will reject it. If you forget to stringify, the server will crash. If you don't check the status, your architecture is fragile. 

Execute perfectly.
