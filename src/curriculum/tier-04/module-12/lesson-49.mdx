When your app needs data, it makes an HTTP Request. The browser has a built-in tool for this called `fetch()`.



### The Request-Response Cycle
1. **Request:** Your frontend sends a message to a URL (a server).
2. **Processing:** The server reads the message, queries its database, and bundles the data.
3. **Response:** The server sends the data back to your frontend over the network.

Because this travels over physical fiber-optic cables, it takes time. Therefore, `fetch()` is always **Asynchronous** (it returns a Promise). 

### Fetching Data (GET)
By default, `fetch()` makes a **GET** request. It is simply asking to "get" data, not change anything.

```javascript
async function getDashboardData() {
  // 1. We make the request and wait for the raw response
  const response = await fetch("[https://api.example.com/data](https://api.example.com/data)");
  
  // 2. The data arrives as a raw text stream. 
  // We must parse it into a JavaScript Object using .json()
  const data = await response.json();
  
  console.log(data);
}
```

*Crucial Rule:* You must `await` twice. Once for the network to return the headers (`fetch`), and once for the body stream to finish downloading and parsing (`.json()`).
