Fetching data is easy. But how do you create a new transaction in *riot-sop*? You can't use a GET request to write to a database. You must use a **POST** request.



### The Options Object
To change `fetch()` from a read-only GET to a write-enabled POST, we pass a second argument: the **Options Object**. 

```javascript
const newTransaction = { amount: 500, type: "deposit" };

const res = await fetch("[https://api.example.com/transactions](https://api.example.com/transactions)", {
  method: "POST",                   // 1. Change the verb
  headers: {
    "Content-Type": "application/json"  // 2. Tell the server what format we are sending
  },
  body: JSON.stringify(newTransaction)  // 3. Attach the actual data (must be a string!)
});
```

### The Three Requirements of POST
1. **The Method:** Must explicitly state `"POST"`.
2. **The Headers:** The server needs to know how to read your data. `"Content-Type": "application/json"` tells the backend, "Hey, I'm sending you JSON, not an image or a PDF."
3. **The Body:** This is the payload. Because networks only transmit text, you cannot send a raw JavaScript object. You must convert it to a string using `JSON.stringify()`.
