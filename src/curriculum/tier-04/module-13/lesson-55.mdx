If `localStorage` is permanent, how do we handle sensitive data?

If you store an authentication token in `localStorage`, it stays there forever. If a user logs in on a public computer and walks away, the next person can open the browser and access their account.

### The Session Sandbox
`sessionStorage` has the **exact same API** as `localStorage` (`setItem`, `getItem`, `removeItem`). 

The only difference is its lifespan.
* `localStorage`: Survives browser restarts. Must be manually deleted.
* `sessionStorage`: **Dies immediately when the tab is closed.** ```javascript
// This data will vanish the second the user closes this specific tab.
sessionStorage.setItem("secure_auth_token", "abc-123");
```

Use `sessionStorage` for temporary state (like multi-page form data) and sensitive session tokens that should not survive a browser restart.

### Your Assignment
You are handling a temporary authentication flow. Store the string `'xyz-987'` in `sessionStorage` under the key `'temp_token'`. Retrieve it and print it to the console to verify it was stored securely in the temporary vault.
