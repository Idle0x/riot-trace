Because `localStorage` only accepts strings, what happens if you try to save a JavaScript object?

```javascript
const node = { id: 1, status: "online" };

// WRONG: JavaScript converts the object to a useless string
localStorage.setItem("nodeData", node); 

console.log(localStorage.getItem("nodeData")); 
// Logs: "[object Object]" <-- Your data is destroyed!
```

### The JSON Solution
To safely store complex data (Objects and Arrays), you must serialize it using `JSON.stringify()`. This turns your entire object into a perfectly formatted text string. 

When you want the data back, you use `JSON.parse()` to rebuild the string back into a living JavaScript object.

```javascript
const node = { id: 1, status: "online" };

// 1. Pack it into a string and save it
const packedData = JSON.stringify(node);
localStorage.setItem("savedNode", packedData);

// 2. Retrieve the string
const rawString = localStorage.getItem("savedNode");

// 3. Unpack it back into a real object
const livingObject = JSON.parse(rawString);

console.log(livingObject.status); // Logs: "online"
```

### Your Assignment
You have a user object. You cannot save it raw. Use the JSON packing method to stringify it, save it under the key `"session_user"`, pull it back out, parse it, and log the user's `role`.
