Every modern browser acts as a mini-database. It has a feature called `localStorage` that allows you to save data directly to the user's hard drive. 

When the user closes the tab, restarts their computer, and comes back a week later, the data is still there.



### The Syntax
The API is incredibly simple. It operates purely on **Key-Value pairs**. 

```javascript
// 1. Saving Data
// Syntax: localStorage.setItem("keyName", "valueData");
localStorage.setItem("username", "Cipher");

// 2. Retrieving Data
const savedName = localStorage.getItem("username");
console.log(savedName); // Logs: "Cipher"

// 3. Deleting Data
localStorage.removeItem("username");
```

### The Catch: Strings Only
`localStorage` is dumb. It can **only** store strings. If you try to pass it an array or a number, it will forcefully convert it into a string, which often corrupts the data. (We will fix this in the next lesson).
