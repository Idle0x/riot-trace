## The Final Gate of Tier 01

You have completed four modules. You have learned what every data type is, how variables and scope work, how JavaScript makes decisions, and how arrays and objects store and expose data.

This capstone does not teach you anything new. It proves that you own what you learned.

The script above is intentionally broken. A junior developer wrote it quickly and introduced 8 bugs — some obvious, some subtle, some that crash the program, some that produce wrong output silently. Your job is to find every single one and fix it without changing the structure of the program.

---

## The Rules

**You may only fix bugs. Do not add new logic, restructure the code, or change how the program works.**

The output must match exactly:

```
[SYS.BOOT] riot-trace v1.2.1
Network: mainnet | Nodes: 201
OPERATIONAL
true
7A5r: 95% — ELITE
TRIAGE COMPLETE
```

Six lines. Exact strings. Correct types.

---

## How to Approach This

Do not try to fix all 8 bugs at once. Work section by section, in order.

**Section 1** — Read the variable name used in the template literal. Is it the same as the constant declared above it?

**Section 2** — The output looks right but the validator will reject it. Check the *type* of `nodeCount`. Run `typeof nodeCount` if you are unsure.

**Section 3** — The `if/else if` chain has a condition at the wrong threshold or in the wrong position. `avgHealth` is 78. Which branch should it match? Trace through the chain top-to-bottom and find where 78 falls.

**Section 4** — There is a right and wrong equality operator in JavaScript. This section uses the wrong one. It produces `true` for this specific case but the habit is incorrect and the validator checks for it.

**Section 5** — Arrays are zero-indexed. The first element is at index 0. The code accesses a specific index — is it the right one for the intended output?

**Section 6** — Two bugs on the same line. One is an operator used for comparison that is actually performing an assignment. One is structural — look at what the compound condition actually evaluates to.

**Section 7 (Bonus)** — `Math.max()` accepts individual numbers as arguments, not an array. The call is passing an array directly. This does not produce a console.log but it does produce `NaN` which can cause downstream issues. Fix it.
