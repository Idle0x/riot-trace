When a new user signs up for *riot-sop*, or a new node connects to *Xandeum Pulse*, you must write that data into the database permanently. 

In SQL, we create new rows using the `INSERT INTO` statement.

### The Syntax Map
Inserting data requires two pairs of parentheses. The first pair targets the **columns**. The second pair holds the actual **values**.

```sql
-- 1. Specify the table and the columns you want to write to
INSERT INTO users (username, role) 
-- 2. Provide the exact values in the exact same order!
VALUES ('Cipher', 'admin');
```

### Auto-Incrementing IDs
Notice that we did not insert an `id` number. In almost all relational databases, the `id` column is set up as a `PRIMARY KEY AUTOINCREMENT`. This means the database acts like an automatic counter. 

When you insert a row, the database automatically assigns it `id: 1`. The next row gets `id: 2`. You never have to manually manage IDs.
