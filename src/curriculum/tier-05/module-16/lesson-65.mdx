When a user closes their account or a temporary node drops off the network forever, you must remove the data to keep your storage costs low and your queries fast.



### The Syntax Map
`DELETE` is simpler than `UPDATE`, which makes it even more dangerous. 

```sql
DELETE FROM transactions WHERE status = 'failed';
```

*Note: You do not specify columns when deleting. You cannot delete a single cell in SQL. `DELETE` removes the **entire row** from existence.*

### Soft Deletes vs Hard Deletes
In real-world applications (like banking or healthcare), we almost never use the `DELETE` command. It is too destructive. 

Instead, we use a **Soft Delete**. We add a column called `is_deleted` and use an `UPDATE` command to set it to `true`. Then, we just modify all our `SELECT` queries to `WHERE is_deleted = false`. The data is hidden from the user, but the engineers still have the backup.

For this lesson, however, we are doing a real, permanent **Hard Delete**.

### Your Assignment
Two nodes have completely dropped off the network.
Write a query to `DELETE FROM` the `nodes` table `WHERE` the `status` is exactly `'offline'`.
