Querying the Foreign Key directly is useful, but what if you want to display the transaction amount AND the username on the same dashboard? 

You need data from both tables at the exact same time. You need a **JOIN**.



### The Syntax Map
An `INNER JOIN` merges rows from two tables into a single mega-row, but *only* if there is a match between the tables.

```sql
SELECT * FROM users 
JOIN transactions 
-- The ON keyword tells the database exactly how the tables connect
ON users.id = transactions.user_id;
```

### Resolving Ambiguity
Notice how we used `users.id` instead of just `id`. 
Because *both* tables have an `id` column (the user has an ID, and the transaction has an ID), if you just type `id`, the database will crash with an "Ambiguous Column" error. 

You must prepend the table name to the column using dot notation when doing JOINs.
