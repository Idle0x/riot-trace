How does a database know that a specific transaction belongs to Cipher, and not to Alice? 

It uses a mathematical link between the tables called **Keys**.



### The Primary Key (PK)
Every table must have one column that uniquely identifies the row. We already saw this: it's the `id` column. No two users can have the same `id`.

### The Foreign Key (FK)
To link a transaction to a user, we add a column to the `transactions` table called `user_id`. 
This column holds the Primary Key of the user it belongs to. Because it references a Primary Key from an *outside* table, it is called a **Foreign Key**.

```sql
-- users table
| id (PK) | username |
|---------|----------|
| 1       | Alice    |
| 42      | Cipher   |

-- transactions table
| id (PK) | amount | user_id (FK) |
|---------|--------|--------------|
| 99      | 500    | 42           | <-- Belongs to Cipher
| 100     | 1000   | 1            | <-- Belongs to Alice
```
