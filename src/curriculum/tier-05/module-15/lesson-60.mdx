In JavaScript, you used `array.filter()` to remove items you didn't want. In SQL, we use the **WHERE** clause. 

It is always faster to filter data *inside the database* rather than sending 10,000 rows over the internet and filtering it on the frontend.

### The Syntax
The `WHERE` clause comes immediately after the `FROM` statement.

```sql
SELECT * FROM users WHERE role = 'admin';
```
*(Crucial Note: In JavaScript, you use `===` for equality and double quotes `" "` for strings. In SQL, you use a single `=` for equality and single quotes `' '` for strings!)*

### Logic Operators
You can use standard mathematical operators to filter numbers, and you can combine conditions using `AND` / `OR`.

```sql
-- Find nodes with low health
SELECT * FROM nodes WHERE health < 50;

-- Find specific active nodes
SELECT * FROM nodes WHERE status = 'online' AND network = 'mainnet';
```
