Arrays in JavaScript are temporary. If the server crashes, the array is destroyed. 
To store mission-critical data permanently, we use a **Relational Database** (like PostgreSQL, MySQL, or SQLite).

In a relational database, data is not stored as loose objects. It is stored in strict, highly organized **Tables**.



### The Anatomy of a Table
Think of a table exactly like a spreadsheet.
* **Table:** The spreadsheet itself (e.g., `nodes`).
* **Columns:** The strict categories of data (e.g., `id`, `name`, `health`).
* **Rows:** The actual individual records (e.g., Node Alpha, Node Beta).

### The SELECT Keyword
You cannot write JavaScript to ask a database for data. You must write **SQL**. 
To read data, you use the `SELECT` statement. 

```sql
-- This gets the specific 'name' and 'health' columns from the 'nodes' table
SELECT name, health FROM nodes;
```

*(Note: In SQL, comments start with `--` instead of `//`)*.

### The Wildcard (*)
If you want to grab *every single column* without typing them all out manually, you use the asterisk `*` (pronounced "star" or "wildcard").

```sql
-- Read as: "Select ALL columns from the users table"
SELECT * FROM users;
```

**Rule of SQL:** Every complete SQL command should end with a semicolon `;`. This tells the engine the query is finished.
