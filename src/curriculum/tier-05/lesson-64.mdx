Data changes. A node goes offline, a transaction clears, a user upgrades their tier. To change data that already exists, we use the `UPDATE` statement.



### The Syntax Map
Updating requires three keywords: `UPDATE`, `SET`, and the incredibly critical `WHERE`.

```sql
UPDATE users 
SET role = 'admin' 
WHERE username = 'Alice';
```

### The Most Dangerous Command in SQL
Look closely at the query above. What happens if you forget the `WHERE` clause and just run this?

```sql
UPDATE users SET role = 'admin';
```
**Catastrophe.** Because you didn't specify *who* to update, the database assumes you meant *everyone*. Every single user in your entire system just became an admin. 

In enterprise environments, forgetting a `WHERE` clause on an `UPDATE` or `DELETE` statement is a fireable offense. Always triple-check your conditions before executing.
