Underneath the hood, a JavaScript Array requires a continuous, unbroken block of physical memory in your RAM. If you need an array of 10 million items, your computer might struggle to find a single, unbroken block of memory that large.

A **Linked List** solves this by scattering data randomly throughout the computer's memory.



### Nodes and Pointers
In a Linked List, every piece of data is wrapped in an object called a `Node`. 
A Node contains two things:
1. **Value:** The actual data.
2. **Next:** A pointer (memory address) telling the computer exactly where to find the next node in the chain.

Because each node points to the next, the data does not need to sit next to each other physically.

### Traversing the Chain
You cannot jump straight to index `[5]` in a Linked List. To find the 5th item, you must start at the 1st item, follow its pointer to the 2nd, follow its pointer to the 3rd, and so on. Searching is always $O(n)$. 

However, inserting a new item at the very beginning of a Linked List is $O(1)$ (Instant), whereas adding an item to the beginning of an Array is $O(n)$ because every other item has to shift its index over by one.
