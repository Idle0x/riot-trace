This is the most common algorithmic failure in junior engineering. 

You have two lists. You need to find the items that match. The natural instinct is to write a loop inside a loop.

### The Danger of Nested Loops
```javascript
// NEVER DO THIS FOR LARGE DATA
for (const nodeA of listA) {
  for (const nodeB of listB) {
    if (nodeA === nodeB) console.log("Match!");
  }
}
```
If `listA` has 10,000 items, and `listB` has 10,000 items, the inner loop runs 10,000 times *for every single item in the outer loop*. 

$10,000 \times 10,000 = 100,000,000$ operations. 

This is **$O(n^2)$ (Quadratic Time)**. The time cost explodes exponentially as the data grows.

### The Set: A Faster Map
To solve this in $O(n)$ time, we use a `Set`. A Set is like an Array, but it is built on a Hash Map structure, meaning checking if an item exists inside a Set is **instant ($O(1)$)**.

```javascript
// 1. Convert the first array into a Set (Takes O(n) time)
const fastLookup = new Set(listA);

// 2. Loop through the second array ONCE (Takes O(n) time)
for (const item of listB) {
  // .has() is an O(1) instant lookup!
  if (fastLookup.has(item)) {
    console.log("Match found:", item);
  }
}
// Total Time: O(n) + O(n) = O(n). We just saved 99 million operations.
```
