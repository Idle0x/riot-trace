If searching an array for a specific user takes $O(n)$ time, how does a database of 1 billion users find your profile instantly when you log in?

It doesn't use an array. It uses a **Hash Map** (also called a Dictionary).

### The Power of Objects
In JavaScript, an Object is a Hash Map. Retrieving a value from an object using its key (e.g., `map["target"]`) is an **$O(1)$ Constant Time** operation. The machine does not have to search; it mathematically calculates the exact memory address of that key.

### Converting $O(n)$ to $O(1)$
If you know you are going to look up data frequently, you should convert your Array into a Map.

```javascript
const usersArray = [
  { id: "u1", name: "Alice" },
  { id: "u2", name: "Bob" }
];

// 1. Create the empty Map
const usersMap = {};

// 2. Populate the Map (This loop takes O(n) time, but we only do it ONCE)
for (const user of usersArray) {
  usersMap[user.id] = user; 
}

/* usersMap now looks like this:
{
  "u1": { id: "u1", name: "Alice" },
  "u2": { id: "u2", name: "Bob" }
}
*/

// 3. INSTANT LOOKUP! (O(1) time forever)
console.log(usersMap["u2"].name); // "Bob"
```
