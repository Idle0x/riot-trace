### EXECUTION REQUIREMENTS
This is a Boss Fight. There are no hints. You must achieve **O(N) Time Complexity**.

You are writing the pairing logic for Xandeum pNodes. You are given an array of available `bandwidths`, and a `target` bandwidth. 
You must find the **two** bandwidths in the array that add up exactly to the target, and return them.

**The Catch:**
If you use a nested loop (checking every number against every other number), my validation engine will feed your function an array of 40,000 nodes. A nested loop will require 1.6 billion operations. The engine will detect the CPU strain, flag a `TIME LIMIT EXCEEDED` error, and fail your build.

**The Strategy:**
1. Loop through the array **exactly once**.
2. For each number, calculate what its "partner" would need to be to hit the target (`target - currentNumber`).
3. Check if you have already seen that partner by looking inside a `Set` or Hash Map (Instant O(1) lookup).
4. If you haven't seen it, store the `currentNumber` in the Set for future numbers to find.
5. If you have seen it, you found your pair! Return them as an array.

Execute perfectly.
