How do we measure if code is "fast"? We don't use seconds, because a supercomputer will always run code faster than an old laptop. 

Instead, we measure **how the number of operations grows as the data grows**. This is called **Big O Notation**.



### $O(1)$: Constant Time (The Holy Grail)
If an operation takes the exact same amount of effort regardless of whether you have 10 items or 10 billion items, it is $O(1)$.

```javascript
// Reading the first item of an array is O(1). 
// The machine knows exactly where index 0 is in memory. 
// It does not have to look at the rest of the array.
const first = nodes[0]; 
```

### $O(n)$: Linear Time (The Standard)
If you have to search through an array one by one to find something, the cost scales linearly with the data size ($n$). If the array doubles in size, the worst-case search time doubles.

```javascript
// This is O(n). In the worst case, the node we want 
// is at the very end, so we must check every single item.
const target = nodes.find(node => node.id === "Alpha");
```
