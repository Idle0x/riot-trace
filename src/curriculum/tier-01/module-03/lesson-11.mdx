Sometimes an `if/else` block takes up too much vertical space. A **Ternary** collapses it to one line. 
A **Switch** statement handles the case where you are checking one specific variable against many exact possibilities.

### The Ternary Operator
```javascript
// condition ? valueIfTrue : valueIfFalse
const health = 95;
const status = health > 80 ? "healthy" : "at risk";

// Read as: "Is health > 80? If yes, 'healthy'. If no, 'at risk'."
```
*Rule of thumb: Never nest ternaries (a ternary inside a ternary). It becomes completely unreadable.*

### The Switch Statement
Cleaner than an `if/else if` chain when checking a single variable against strict values.

```javascript
const network = "mainnet";

switch (network) {
  case "mainnet":
    console.log("Production network");
    break;  // BREAK IS REQUIRED! 
  case "devnet":
    console.log("Development network");
    break;
  default:
    console.log("Unknown network");
}
```
**The Fallthrough Bug:** If you forget the `break;` keyword, JavaScript will execute the matched case, *and then continue executing every case below it* until it hits a break or the end of the switch.
