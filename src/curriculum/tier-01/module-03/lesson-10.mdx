A node is "at risk" if health is low AND it's on mainnet. A user can access a dashboard if they're logged in OR they're an owner. Real conditions are rarely single checks. 



### The Combiners

**1. AND (`&&`)**
BOTH conditions must be true for the block to execute.
```javascript
const health = 65;
const network = "mainnet";

if (health < 70 && network === "mainnet") {
  console.log("Mainnet node at risk");  // logs — both conditions are true
}
```

**2. OR (`||`)**
AT LEAST ONE condition must be true.
```javascript
const isAdmin = false;
const isOwner = true;

if (isAdmin || isOwner) {
  console.log("Access granted");  // logs — isOwner is true
}
```

**3. NOT (`!`)**
Inverts a boolean value. `!true` becomes `false`.
```javascript
const isOffline = false;
if (!isOffline) {
  console.log("Node is reachable");  // logs — !false = true
}
```

### Short-Circuit Evaluation
JavaScript is lazy (in a good way). It stops reading the moment it knows the answer:
* `A && B` — if A is false, JS doesn't even bother looking at B. It already knows the whole statement can't be true.
* `A || B` — if A is true, JS stops. It already knows the statement is true.
