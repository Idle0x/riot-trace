Is the user logged in? Is this node healthy? Is the XP above zero? Every conditional, every filter, every auth check — all of it reduces to `true` or `false`. This is the foundation of all decision-making in code.

### The Boolean
A boolean has exactly two possible values. They are not strings; do not wrap them in quotes.
```javascript
true    
false   
```

### Comparison Operators
These operators compare two values and **always** produce a boolean result.
```javascript
5 > 3           // true
5 < 3           // false
5 >= 5          // true  (greater than OR equal)
5 <= 4          // false
```

### The Golden Rule: `===` vs `==`
This is critical.
* `==` (Loose Equality): Converts types before comparing. This is dangerous and causes unpredictable bugs. Example: `0 == false` returns `true`.
* `===` (Strict Equality): Never converts types. It checks both VALUE and TYPE. Example: `0 === false` returns `false`.

**RULE: Always use `===`. Never use `==`.**

```javascript
5 === 5         // true
"5" === 5       // false (String vs Number)
true === true   // true
"true" === true // false (String vs Boolean)
```
