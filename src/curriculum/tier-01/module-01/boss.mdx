## The Test

You have completed four lessons. You have learned what strings, numbers, booleans, and empty values are at the type level. You understand `console.log`, the mathematical operators, strict equality, and `typeof`.

Now you prove it without assistance.

The panel on the right contains a blank terminal with only comments explaining the mission. There is no code to modify. There is no scaffold to lean on. You write all four lines from scratch.

Read the requirements carefully. The validator is strict — it checks the type, the value, and the position of every output line. A string where a number is expected fails. A boolean where a string is expected fails. An extra empty line fails.

---

## What the Validator Is Checking

Four outputs. Four checks. Here is what each line must be, in plain terms:

**Line 1** — A string. It starts with a bracket, contains text, and ends with three period characters followed by a closing bracket. This is the exact format used in professional monitoring system logs. One wrong character and it fails.

**Line 2** — A number. The result of a multiplication operation. Not the hardcoded result — the actual expression. `console.log(8192)` is wrong even if it produces the right number. The validator checks that your code computes it, not that you computed it for your code.

**Line 3** — A boolean. The result of a strict equality comparison between two strings. Do not log the strings themselves. Log the comparison.

**Line 4** — A string. The type name of the result from Line 3. You learned in Lesson 3 that `typeof` returns a string. That string is what belongs on Line 4.
