XP counts, node health scores, storage in terabytes, credit balances in riot-sop — your apps are full of numbers. This lesson makes you fluent in how JavaScript calculates data.

### Number Types
Unlike strings, numbers do **not** get wrapped in quotes. If you wrap a number in quotes (like `"42"`), the machine treats it as text, not math.

JavaScript has one `number` type for both integers (whole numbers) and floats (decimals). 

```javascript
// Integers
42
201         // nodes on the network

// Floats
429.47      // TB of network capacity
78.3        // average health score
```

### Mathematical Operators
The machine calculates exactly like a calculator, using specific symbols:



```javascript
10 + 5      // 15  — Addition
10 - 5      // 5   — Subtraction
10 * 5      // 50  — Multiplication
10 / 5      // 2   — Division
10 % 3      // 1   — Modulo (Returns the remainder after division)
```
*Note: Modulo (`%`) is incredibly useful in programming. For example, if you want to know if a number is even or odd, you just check if `number % 2` leaves a remainder of `0`.*

### The `typeof` Operator
Sometimes you receive data from an API and you don't know what it is. The `typeof` operator asks the machine to identify the data. It always returns a string telling you the data type.

```javascript
typeof 42           // returns "number"
typeof "hello"      // returns "string"
```
