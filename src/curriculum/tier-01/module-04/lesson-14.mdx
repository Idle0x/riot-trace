Every node in xandeum-pulse is an object. Every transaction in riot-sop is an object. Objects are the single most important data structure in JavaScript.



### Key-Value Pairs
An object holds data not by order (like an array), but by **labels (keys)**.
```javascript
const node = {
  id: "7A5rRdb",
  health: 95,
  network: "mainnet",
  isOnline: true
};
```
* **Keys** (left of colon) — think of them as property names.
* **Values** (right of colon) — can be any data type (string, number, boolean, array, or even another object).

### Accessing & Modifying Data
There are two ways to read or change data in an object.

**1. Dot Notation (Preferred)**
Used when you know the exact name of the key you want.
```javascript
node.health = 87;      // Modifies existing property
node.version = "1.0";  // Adds a completely new property
console.log(node.network); // Reads property
```

**2. Bracket Notation**
Used when the key is dynamic (stored inside a variable) or contains invalid characters like spaces.
```javascript
const targetKey = "health";
console.log(node[targetKey]); // Evaluates to node["health"] -> 95
```
