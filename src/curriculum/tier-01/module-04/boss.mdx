## The Test

Module 04 covered the two most important data structures in JavaScript. Arrays for ordered lists. Objects for structured descriptions. Destructuring and spread for extracting and composing data cleanly.

This boss fight combines all of them in a single realistic scenario: processing a raw API response the same way your actual code does in xandeum-pulse.

---

## What This Tests

**Requirement 1** tests nested destructuring — going two levels deep in one or two lines to extract `timestamp`, `network`, and `nodes` without using dot-chained assignment.

**Requirement 2** tests combining array and object destructuring in sequence. You destructure the first element from the array (`const [firstNode] = nodes`), then destructure properties from that object (`const { id, health } = firstNode`). These can be chained or written separately.

**Requirement 3** introduces `Array.prototype.find()` — the version of `indexOf` that works on objects. Unlike `includes()` and `indexOf()` which compare by reference or exact value, `find()` accepts a callback that tests each element:

```javascript
nodes.find(n => n.id === "8BJT")
// Returns the first node object where n.id === "8BJT"
// Returns undefined if no match is found
```

You will use `.find()` constantly starting in Tier 2. This is your first exposure to it.

**Requirement 4** tests spread for object composition — taking an existing object and adding new properties in one expression. The `network` property must come from the variable you destructured in Requirement 1, not hardcoded.

**Requirement 5** tests the `in` operator for property existence checking before accessing a value.

---

## Approach

Work through each requirement in order. Requirement 1's variables feed into Requirements 2, 3, and 4. Do not skip ahead.

If Requirement 3 is unfamiliar — `.find()` takes a function and returns the first element for which that function returns `true`. This is a preview of Tier 2's array methods module. Read the hint in the requirements carefully.
