## Why Arrays Are Everywhere

Open your xandeum-pulse codebase. Find the API response from the stats endpoint. It returns something like this:

```javascript
{
  pods: [
    { id: "7A5r", health: 95, network: "mainnet" },
    { id: "8BJT", health: 47, network: "devnet" },
    { id: "8PjP", health: 78, network: "mainnet" },
    // ... 198 more nodes
  ],
  totalCount: 201
}
```

That `pods` property is an array. Every node card rendered on the dashboard comes from iterating over that array. Every filter button modifies which items from that array are shown. Every leaderboard is that array sorted differently.

Arrays are the primary container for any list of data in JavaScript. Understanding them at this level — not just "how to use them" but "what they are, how they work, what every method does" — is the difference between reading data processing code fluently and squinting at it line by line.

---

## What an Array Actually Is

An array is an **ordered, indexed collection of values**. Ordered means the sequence matters — item 0 is always item 0. Indexed means every item has a position number starting at 0.

```javascript
const nodes = ["node-A", "node-B", "node-C"];
//              index 0    index 1    index 2
```

An array can hold any mix of data types:

```javascript
const mixed = [42, "mainnet", true, null, { health: 95 }];
// Valid JavaScript — but in practice, arrays hold one type per array
// Mixed-type arrays are harder to work with and a code smell
```

In real codebases, every array holds the same type of thing: all strings, all numbers, all node objects. TypeScript enforces this with type annotations, which you will use throughout Tiers 2 and 3.

## Creating Arrays

```javascript
// Array literal — the standard way
const tiers = ["Code Literacy", "JavaScript Deeply", "React Mental Model"];

// Empty array — common when you will add items later
const activeNodes = [];

// Array constructor — rarely used, prefer literal syntax
const scores = new Array(3);  // creates [undefined, undefined, undefined]
// This is confusing and almost never what you want
```

Always use the literal `[]` syntax. The constructor form has subtle edge cases that make it unreliable.

## Accessing Elements

Arrays are zero-indexed. The first item is at index `0`, not index `1`:

```javascript
const tiers = ["Code Literacy", "JavaScript Deeply", "React Mental Model"];

tiers[0]   // "Code Literacy" — first item
tiers[1]   // "JavaScript Deeply" — second item
tiers[2]   // "React Mental Model" — third and last item
tiers[3]   // undefined — index 3 does not exist, no error, just undefined
tiers[-1]  // undefined — negative indices don't work in JavaScript the way you might expect
```

**Accessing the last element dynamically** — never hardcode the last index:

```javascript
// Bad — hardcoded index breaks if the array grows:
const last = tiers[2];

// Good — always works regardless of array length:
const last = tiers[tiers.length - 1];

// Modern alternative (ES2022):
const last = tiers.at(-1);  // .at() supports negative indices
```

## The length Property

`length` returns the number of elements in the array. It is a property, not a method — no parentheses:

```javascript
const nodes = ["node-A", "node-B", "node-C"];
nodes.length   // 3

const empty = [];
empty.length   // 0
```

`length` is always one more than the last valid index:

```javascript
const arr = ["a", "b", "c"];
// Last valid index: 2
// arr.length: 3
// arr[arr.length - 1]: "c"
```

## Adding and Removing Elements

Four methods. Two add, two remove. Learn all four:

```javascript
const nodes = ["node-A", "node-B", "node-C"];

// push() — adds to the END, returns new length
nodes.push("node-D");
// nodes: ["node-A", "node-B", "node-C", "node-D"]
// return value: 4 (new length)

// unshift() — adds to the BEGINNING, returns new length
nodes.unshift("node-Z");
// nodes: ["node-Z", "node-A", "node-B", "node-C", "node-D"]

// pop() — removes from the END, returns the removed item
const removed = nodes.pop();
// removed: "node-D"
// nodes: ["node-Z", "node-A", "node-B", "node-C"]

// shift() — removes from the BEGINNING, returns the removed item
const first = nodes.shift();
// first: "node-Z"
// nodes: ["node-A", "node-B", "node-C"]
```

**Memory trick:**
- `push` / `pop` — both reference the END (like a stack)
- `shift` / `unshift` — both reference the BEGINNING (un-shift puts something back at front)

All four **mutate the original array** — they change it in place. This matters for React state management, which you will learn in Tier 3.

## Finding Elements

```javascript
const nodes = ["node-A", "node-B", "node-C", "node-D"];

// includes() — does the item exist? Returns boolean
nodes.includes("node-B")    // true
nodes.includes("node-Z")    // false

// indexOf() — what position is the item at? Returns index or -1
nodes.indexOf("node-B")     // 1
nodes.indexOf("node-Z")     // -1 (not found)

// Finding -1 is how you check for absence:
if (nodes.indexOf("node-Z") === -1) {
  console.log("node-Z is not in the array");
}
// Or more cleanly with includes():
if (!nodes.includes("node-Z")) {
  console.log("node-Z is not in the array");
}
```

## Slicing — Non-Destructive Extraction

`slice()` returns a new array containing a portion of the original. The original is not modified:

```javascript
const scores = [95, 47, 88, 71, 55, 91];

scores.slice(0, 3)    // [95, 47, 88] — items at index 0, 1, 2 (3 is exclusive)
scores.slice(2)       // [88, 71, 55, 91] — from index 2 to end
scores.slice(-2)      // [55, 91] — last 2 items
scores.slice(1, 4)    // [47, 88, 71]

console.log(scores);  // [95, 47, 88, 71, 55, 91] — unchanged
```

`slice()` is useful for getting the "top N" from a sorted list:

```javascript
const sortedNodes = [...nodes].sort((a, b) => b.health - a.health);
const topFive = sortedNodes.slice(0, 5);  // top 5 healthiest
```

## Splicing — Destructive Modification

`splice()` modifies the array in place — it can remove items, insert items, or both:

```javascript
const nodes = ["node-A", "node-B", "node-C", "node-D"];

// splice(startIndex, deleteCount, ...itemsToInsert)
nodes.splice(1, 2);
// Removes 2 items starting at index 1 (removes "node-B" and "node-C")
// nodes: ["node-A", "node-D"]
// Returns: ["node-B", "node-C"] (the removed items)

// Insert without removing:
nodes.splice(1, 0, "node-X");
// At index 1, remove 0 items, insert "node-X"
// nodes: ["node-A", "node-X", "node-D"]
```

`splice()` mutates. `slice()` does not. This is one of JavaScript's most confusing naming decisions. The difference is the `p` — `splice` with a `p` is destructive.

## Joining and Splitting

```javascript
const tiers = ["Code Literacy", "JavaScript Deeply", "React Mental Model"];

// join() — array to string, items separated by delimiter
tiers.join(", ")      // "Code Literacy, JavaScript Deeply, React Mental Model"
tiers.join(" → ")     // "Code Literacy → JavaScript Deeply → React Mental Model"
tiers.join("")        // "Code LiteracyJavaScript DeeplyReact Mental Model"

// The reverse: splitting a string into an array
const csv = "mainnet,devnet,testnet";
csv.split(",")        // ["mainnet", "devnet", "testnet"]
```

## Copying Arrays — Reference Trap Revisited

From Lesson 5, you know that objects and arrays are reference types. This matters here:

```javascript
const original = [1, 2, 3];
const reference = original;    // NOT a copy — same array in memory
reference.push(4);
console.log(original);         // [1, 2, 3, 4] — original changed!

// Shallow copy — three equivalent ways:
const copy1 = [...original];          // spread operator (most common)
const copy2 = original.slice();       // slice with no arguments
const copy3 = Array.from(original);   // Array.from

copy1.push(99);
console.log(original);         // [1, 2, 3, 4] — original unchanged
console.log(copy1);            // [1, 2, 3, 4, 99] — independent
```

In React, you almost always want a copy when modifying array state. Mutating the original array directly often causes the UI not to update. This is why `[...array]` appears constantly in React codebases.

## Checking If Something Is an Array

The `typeof` operator is useless for arrays — it returns `"object"` for everything:

```javascript
typeof [1, 2, 3]    // "object" — not helpful
typeof {}           // "object" — same result

// Use Array.isArray() instead:
Array.isArray([1, 2, 3])   // true
Array.isArray({})           // false
Array.isArray("string")     // false
Array.isArray(null)         // false
```

## Common Mistakes

**Mistake 1 — Off-by-one indexing:**
```javascript
const arr = ["a", "b", "c"];
arr[3]    // undefined — last valid index is 2 (length - 1)
arr[arr.length]  // undefined — same mistake
arr[arr.length - 1]  // "c" — correct last item
```

**Mistake 2 — Confusing slice and splice:**
```javascript
const arr = [1, 2, 3, 4, 5];
arr.slice(1, 3)    // [2, 3] — non-destructive, arr unchanged
arr.splice(1, 2)   // removes and returns [2, 3], arr is now [1, 4, 5]
```

**Mistake 3 — Mutating instead of copying:**
```javascript
const nodes = ["node-A", "node-B"];
const updated = nodes;         // same array
updated.push("node-C");
console.log(nodes.length);     // 3 — original mutated!

// Fix:
const updated = [...nodes];    // copy
updated.push("node-C");
console.log(nodes.length);     // 2 — original safe
```

**Mistake 4 — Using .size instead of .length:**
```javascript
const arr = [1, 2, 3];
arr.size      // undefined — arrays don't have .size
arr.length    // 3 — correct
// Note: JavaScript Sets and Maps DO have .size. Easy to confuse.
```

**Mistake 5 — Expecting push() to return the array:**
```javascript
const result = [1, 2].push(3);
console.log(result);  // 3 — push returns the NEW LENGTH, not the array
// To get the modified array:
const arr = [1, 2];
arr.push(3);
console.log(arr);  // [1, 2, 3]
```
