## The Syntax That Unlocks Modern JavaScript

Every time you use `useState` in React, you write this:

```javascript
const [count, setCount] = useState(0);
```

Every time you fetch data from Supabase, you write this:

```javascript
const { data, error } = await supabase.from("nodes").select("*");
```

Every time you write a React component, you use this:

```javascript
function NodeCard({ id, health, network }) { }
```

All three of these are **destructuring**. If you have been using React, you have been writing destructuring every single day. After this lesson, you will understand exactly what it does and why — which means you will stop being confused when it behaves unexpectedly.

Spread (`...`) appears alongside destructuring constantly. Together they are arguably the two most important syntax features introduced in modern JavaScript.

## Object Destructuring — The Full Picture

Destructuring extracts values from an object into named variables. The long way:

```javascript
const node = { id: "7A5r", health: 95, network: "mainnet" };

const id = node.id;
const health = node.health;
const network = node.network;
```

The destructuring shorthand:

```javascript
const { id, health, network } = node;
// Identical result. One line instead of three.
```

The `{ }` on the left side of `=` is the destructuring pattern. Each name inside must match a property name on the object on the right side.

---

## Renaming While Destructuring

When you need the variable to have a different name than the property:

```javascript
const node = { id: "7A5r", health: 95, version: "1.2.1" };

const { version: nodeVersion } = node;
// Extracts node.version into a variable called nodeVersion
// No variable called 'version' is created

console.log(nodeVersion);   // "1.2.1"
// console.log(version);   // ReferenceError — 'version' doesn't exist
```

The syntax reads: "from the object, take the property called `version` and store it in a variable called `nodeVersion`."

This pattern is common when:
- A property name conflicts with an existing variable
- You want a more descriptive local name
- You are destructuring two objects that have the same property names

---

## Default Values in Destructuring

When a property doesn't exist on the object, it returns `undefined`. You can provide a fallback:

```javascript
const node = { id: "7A5r", health: 95 };

const { health, uptime = 0, version = "unknown" } = node;
// health: 95 (exists in node)
// uptime: 0 (doesn't exist in node, uses default)
// version: "unknown" (doesn't exist in node, uses default)
```

Defaults only apply when the value is `undefined`. An explicit `null` does not trigger the default:

```javascript
const { version = "unknown" } = { version: null };
// version: null — null is not undefined, so default doesn't apply
```

---

## Combining Rename and Default

Both features work together:

```javascript
const { version: nodeVersion = "1.0.0" } = node;
// Take the 'version' property, call it 'nodeVersion',
// and use "1.0.0" if version doesn't exist
```

## Nested Object Destructuring

You can destructure inside nested objects:

```javascript
const node = {
  id: "7A5r",
  location: {
    city: "Tokyo",
    country: "Japan"
  }
};

// Without nested destructuring:
const city = node.location.city;

// With nested destructuring:
const { location: { city, country } } = node;
console.log(city);     // "Tokyo"
console.log(country);  // "Japan"
// Note: 'location' itself is NOT extracted as a variable here
// Only city and country are
```

Use nested destructuring when you need multiple properties from a nested object. If you only need one, a simple property access chain (`node.location.city`) is cleaner.

## Array Destructuring

The same concept applied to arrays — instead of property names, you use position:

```javascript
const tiers = ["Code Literacy", "JavaScript Deeply", "React Mental Model"];

const [first, second, third] = tiers;
// first:  "Code Literacy"
// second: "JavaScript Deeply"
// third:  "React Mental Model"
```

Array destructuring uses `[]` instead of `{}`. The order of variables matches the order of items in the array.

**Skipping elements:**

```javascript
const scores = [95, 47, 88, 71, 55];
const [, second,, fourth] = scores;  // empty commas skip positions
// second: 47
// fourth: 71
```

**The `useState` pattern explained:**

```javascript
const [count, setCount] = useState(0);
// useState(0) returns an array: [currentValue, setterFunction]
// Array destructuring unpacks them into count and setCount
// This is exactly: const result = useState(0); const count = result[0]; const setCount = result[1];
```

## The Rest Operator in Destructuring

The `...rest` syntax collects everything not explicitly destructured:

```javascript
// In objects:
const node = { id: "7A5r", health: 95, network: "mainnet", version: "1.2.1" };
const { id, health, ...rest } = node;
// id: "7A5r"
// health: 95
// rest: { network: "mainnet", version: "1.2.1" }

// In arrays:
const scores = [95, 47, 88, 71, 55];
const [first, second, ...remaining] = scores;
// first: 95
// second: 47
// remaining: [88, 71, 55]
```

The rest variable must always be **last** in the pattern. Putting it anywhere else is a syntax error.

```javascript
const { ...rest, id } = node;  // SyntaxError — rest must come last
const { id, ...rest } = node;  // Valid
```

## The Spread Operator — Expanding Collections

The same `...` syntax used in a different position **expands** a collection into individual items:

```javascript
// Expanding an array into function arguments:
const scores = [95, 47, 88];
Math.max(...scores)     // same as Math.max(95, 47, 88) → 95

// Expanding into a new array:
const a = [1, 2, 3];
const b = [4, 5, 6];
const combined = [...a, ...b];  // [1, 2, 3, 4, 5, 6]
const withExtra = [0, ...a, 99]; // [0, 1, 2, 3, 99]

// Copying an array:
const copy = [...a];  // [1, 2, 3] — new array, independent from a

// Expanding into an object:
const node = { id: "7A5r", health: 95 };
const updated = { ...node, health: 87, version: "1.2.1" };
// { id: "7A5r", health: 87, version: "1.2.1" }
// health from node (95) is overridden by health in the literal (87)
// version is added as a new property
```

**The override rule for object spread:** Later properties win. The rightmost value for any key is what survives:

```javascript
const base = { a: 1, b: 2, c: 3 };
const override = { b: 99, d: 4 };

{ ...base, ...override }   // { a: 1, b: 99, c: 3, d: 4 }
// b from override (99) wins over b from base (2)

{ ...override, ...base }   // { a: 1, b: 2, c: 3, d: 4 }
// b from base (2) wins — base spread came last
```

This is why in React state updates, you spread the existing state first and put the new value after:

```javascript
// Correct — new value overrides the old:
setConfig({ ...currentConfig, timeout: 10000 });

// Wrong — currentConfig overrides your new value:
setConfig({ timeout: 10000, ...currentConfig });
```

## Destructuring in Function Parameters

You can destructure directly in a function's parameter list:

```javascript
// Without destructuring:
function displayNode(node) {
  console.log(node.id, node.health, node.network);
}

// With destructuring in parameters:
function displayNode({ id, health, network }) {
  console.log(id, health, network);
}

// Both are called identically:
displayNode({ id: "7A5r", health: 95, network: "mainnet" });
```

React components use this pattern for props:

```javascript
// Every React component prop you have ever seen is object destructuring:
function NodeCard({ id, health, network, onSelect }) {
  return (
    <div onClick={() => onSelect(id)}>
      {id}: {health}% on {network}
    </div>
  );
}
```

The component receives a single `props` object. The destructuring in the parameter list unpacks it immediately into named variables.

## Common Mistakes

**Mistake 1 — Wrong side of assignment for destructuring:**
```javascript
const { id, health } = node;   // correct — destructuring on LEFT of =
{ id, health } = const node;   // SyntaxError — backwards
```

**Mistake 2 — Forgetting that order matters in object spread:**
```javascript
const merged = { ...base, ...updates };  // updates override base
const merged = { ...updates, ...base };  // base overrides updates
// Which do you want? The one that comes LAST wins.
```

**Mistake 3 — Using rest before the last position:**
```javascript
const { ...rest, id } = node;    // SyntaxError
const { id, ...rest } = node;    // Valid
```

**Mistake 4 — Destructuring a null or undefined value:**
```javascript
const response = null;
const { data } = response;  // TypeError — can't destructure null
// Fix: check first, or use default:
const { data } = response || {};  // safe — if response is null, use {}
const { data } = response ?? {};  // better — only triggers for null/undefined
```

**Mistake 5 — Thinking spread creates a deep copy:**
```javascript
const node = { id: "7A5r", location: { city: "Tokyo" } };
const copy = { ...node };
copy.location.city = "Osaka";   // modifies the shared nested object
console.log(node.location.city); // "Osaka" — original affected
// { ...node } only copies top-level properties — nested objects are still shared
```
