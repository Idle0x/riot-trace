You've probably called a function before you defined it in a file, and it somehow worked. Or you tried to use a variable too early and got a crash. **Hoisting** is why. 

### The Two-Pass Engine
The JavaScript engine does not just read top-to-bottom once. It runs in TWO passes:
* **Pass 1:** Scan the entire file, "hoist" (lift) function declarations to the absolute top of their scope in memory.
* **Pass 2:** Execute the code line by line.

### How it affects your code

**1. Function Declarations are fully hoisted**
Because of Pass 1, you can call them before they appear in the file.
```javascript
sayHello();  // ✓ This works even though the definition is below!

function sayHello() {
  console.log("Hello");
}
```

**2. `let` and `const` are NOT hoisted**
They live in a "Temporal Dead Zone" until the engine reaches the exact line they are defined on.
```javascript
// console.log(y);  // ✗ ReferenceError: Cannot access 'y' before initialization
let y = 5;
```

This is another reason we use `let` and `const` instead of the old `var`. `var` is technically hoisted, but given a silent `undefined` value, which leads to invisible bugs. `let` and `const` loudly throw an error, forcing you to fix your architecture immediately.
