You've used `const`, `let`, and `var` without fully knowing why you chose each. After this lesson, you'll never guess again.

Variables are not "boxes" that hold things. They are **labels** that point to a location in memory.

### The Keywords

**1. `const` (Constant)**
The label cannot be reassigned to point to something else. Use this by default for almost everything.
```javascript
const NETWORK = "mainnet";
// NETWORK = "devnet";  ← TypeError: Assignment to constant variable
```

**2. `let` (Mutable)**
The label CAN be reassigned. Use this for counters or values you expect to change over time.
```javascript
let score = 0;
score = 100;  // ✓ This is perfectly fine
```

**3. `var` (Legacy)**
Function-scoped, hoisted weirdly, and largely obsolete. **Do not use it.** It causes scope bugs that `const` and `let` were specifically invented to fix.
