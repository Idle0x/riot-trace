You've used `const`, `let`, and `var` before without fully knowing why you chose each. After this lesson, you will never guess again.

Variables are not "boxes" that hold things. They are **labels** that point to a location in your computer's memory.

[attachment_0](attachment)

### The Three Keywords

**1. `const` (Constant)**
The label cannot be reassigned to point to something else. Use this by default for almost everything. It makes your code predictable.
```javascript
const NETWORK = "mainnet";
// NETWORK = "devnet";  ← TypeError: Assignment to constant variable
```

**2. `let` (Mutable)**
The label CAN be reassigned. Use this for counters or values you expect to change over time (like a user's XP).
```javascript
let score = 0;
score = 100;  // ✓ This is perfectly fine. We reassigned the label.
```

**3. `var` (Legacy)**
This is the old way. It is function-scoped, hoists weirdly, and is largely obsolete. **Do not use it.** It causes bugs that `const` and `let` were specifically invented to fix.
