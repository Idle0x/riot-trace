Ever seen the mysterious `ReferenceError: x is not defined` in your console? Scope explains exactly why certain variables are visible in some places and invisible in others.



### The Three Scopes

**1. GLOBAL SCOPE**
Accessible everywhere. Defined at the absolute top of your file, outside of any functions or blocks.
```javascript
const globalConfig = "mainnet";
```

**2. FUNCTION SCOPE**
Variables declared inside a function *only exist inside that function*. They die the exact millisecond the function finishes running.
```javascript
function processNode() {
  const localHealth = 95;  // lives here, dies here
  console.log(localHealth);  // ✓ Works perfectly
}

// console.log(localHealth);  // ✗ ReferenceError: localHealth is not defined
```

**3. BLOCK SCOPE**
Variables declared inside `{ curly braces }` (like an `if` statement or a loop) using `let` or `const` only exist inside those braces.
```javascript
if (true) {
  const blockVar = "only here";
}
// console.log(blockVar);   // ✗ ReferenceError
```

### The Rule of Visibility
Looking **outward** (up the chain) always works. Code inside a function can see global variables.
Looking **inward** (down the chain) never works. Global code cannot see inside a function.
