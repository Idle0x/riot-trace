## From Reading to Executing

Tier 01 was about reading code and understanding what it means. You learned data types, variables, conditionals, and data structures. You can look at a piece of JavaScript and say precisely what every line does.

Tier 02 is about writing code that does things. The first thing programs do — the thing that separates a list of values from a useful system — is **iteration**: doing something to every item in a collection, one at a time.

Your xandeum-pulse dashboard does not display 201 nodes by writing 201 lines of code. It loops over the array and processes each node in turn. Your riot-sop financial summary does not add 30 transactions manually. It iterates and accumulates. Everything in application development that works with data uses iteration.

This lesson covers the manual approach — the explicit `for` loop. The next four lessons cover the modern functional alternatives that you will use far more often. You need both: the functional methods because they are cleaner and more expressive, and the `for` loop because it is the foundation that makes those methods understandable.

## The Standard For Loop — Complete Anatomy

```javascript
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Output: 0, 1, 2, 3, 4
```

Three parts inside the parentheses, separated by semicolons:

```javascript
for (initialization; condition; update) {
  // body — runs each iteration
}
```

**Initialization** — `let i = 0`
Runs once before the loop starts. Creates the loop counter variable.
Always use `let`, not `const` — `i` changes each iteration.

**Condition** — `i < 5`
Evaluated before each iteration. If `true`, the body runs. If `false`, the loop stops.
When the loop starts, `i` is 0. `0 < 5` is true. Body runs.
After the update, `i` is 1. `1 < 5` is true. Body runs.
...eventually `i` is 5. `5 < 5` is false. Loop stops.

**Update** — `i++`
Runs after each iteration. `i++` is shorthand for `i = i + 1`.
The update runs after the body, before the next condition check.

## Looping Over Arrays

The most common use case: iterating over every element in an array:

```javascript
const nodes = ["node-A", "node-B", "node-C"];

for (let i = 0; i < nodes.length; i++) {
  console.log(nodes[i]);
}
// node-A
// node-B
// node-C
```

`nodes[i]` accesses the element at position `i`. As `i` goes from 0 to `nodes.length - 1`, you visit every element.

**Always use `nodes.length` in the condition**, not a hardcoded number. If the array grows, the loop automatically handles the new elements:

```javascript
// Bad — breaks if array changes:
for (let i = 0; i < 3; i++) { }

// Good — always correct:
for (let i = 0; i < nodes.length; i++) { }
```

## The for...of Loop — Clean Iteration Without Index

When you just need each element and do not need the index, `for...of` is cleaner:

```javascript
const nodes = ["node-A", "node-B", "node-C"];

for (const node of nodes) {
  console.log(node);
}
// node-A
// node-B
// node-C
```

No counter variable. No `[i]` access. Each iteration, `node` is the current element.

Use `const` in `for...of` — the loop variable is a new binding each iteration, not a reassigned one. TypeScript will also correctly infer the type of `node` from the array's type.

**When to use each:**
- `for...of` — when you need each element but not its index
- Standard `for` — when you need the index (for position labels, accessing neighboring elements, backwards iteration)

## The for...in Loop — Object Keys

`for...in` iterates over the **keys** of an object:

```javascript
const node = { id: "7A5r", health: 95, network: "mainnet" };

for (const key in node) {
  console.log(key, node[key]);
}
// id 7A5r
// health 95
// network mainnet
```

Use `for...in` for objects when you need to dynamically process every property. However, `Object.entries()` with `for...of` is usually cleaner for this:

```javascript
for (const [key, value] of Object.entries(node)) {
  console.log(`${key}: ${value}`);
}
```

**Never use `for...in` on arrays.** It iterates over indices as strings and can include inherited properties. Always use `for...of` or a standard `for` loop for arrays.

## break and continue — Controlling the Loop

**`break`** — exits the loop immediately. Nothing after it in the current iteration runs. The loop ends:

```javascript
const scores = [95, 88, 40, 71, 91];

for (const score of scores) {
  if (score < 50) {
    console.log("Critical score found — halting");
    break;    // stops the loop entirely
  }
  console.log(score);
}
// 95
// 88
// Critical score found — halting
// (71 and 91 never reached)
```

**`continue`** — skips the rest of the current iteration and moves to the next one. The loop continues:

```javascript
const scores = [95, 47, 88, 71, 55, 91];

for (const score of scores) {
  if (score < 70) continue;    // skip this score, go to next
  console.log(score);
}
// 95
// 88
// 71
// 91
// (47 and 55 were skipped)
```

## Counting and Accumulating with For Loops

For loops are the foundation of counting and aggregation — even though in Tier 2 you will learn `.reduce()` which handles this more elegantly:

```javascript
const healthScores = [95, 47, 88, 71, 55, 91, 62, 78];

// Count scores above a threshold:
let healthyCount = 0;
for (const score of healthScores) {
  if (score > 80) healthyCount++;
}
console.log(`Healthy nodes: ${healthyCount}`);  // 3

// Sum all scores:
let total = 0;
for (const score of healthScores) {
  total += score;  // shorthand for: total = total + score
}
const average = total / healthScores.length;
console.log(`Average health: ${average.toFixed(1)}`);  // 73.4

// Find the maximum:
let max = healthScores[0];
for (const score of healthScores) {
  if (score > max) max = score;
}
console.log(`Best health: ${max}`);  // 95
```

Each pattern — count, sum, find — shows the same structure: initialize a variable outside the loop, update it inside the loop, use it after the loop.

## Nested Loops

A loop inside a loop. Each iteration of the outer loop runs the entire inner loop:

```javascript
const networks = ["mainnet", "devnet"];
const statuses = ["online", "offline"];

for (const network of networks) {
  for (const status of statuses) {
    console.log(`${network} — ${status}`);
  }
}
// mainnet — online
// mainnet — offline
// devnet — online
// devnet — offline
```

Nested loops are O(n²) — with large arrays they get slow fast. Lesson 79 in Tier 6 covers this in detail. For now, use them when you genuinely need to compare or combine two collections. Avoid them when one loop would work.

## The Infinite Loop — The Loop You Must Never Write

A loop whose condition never becomes `false` runs forever and crashes the browser or Node process:

```javascript
// Never do this:
for (let i = 0; i >= 0; i++) {
  console.log(i);  // i is always >= 0, condition never false
}

// Also never do this:
while (true) {
  // without a break, this runs forever
}
```

Signs your loop might be infinite:
- The condition variable is never updated
- The update moves `i` in the wrong direction (`i++` when condition is `i > 0`)
- The condition can never become `false` for the given data

If your browser tab freezes while testing a loop, force-close it, fix the loop, and reload.

## Common Mistakes

**Mistake 1 — Off-by-one in the condition:**
```javascript
const arr = [1, 2, 3];
// Wrong — i <= arr.length includes index 3 which doesn't exist:
for (let i = 0; i <= arr.length; i++) {
  console.log(arr[i]);  // logs 1, 2, 3, undefined
}
// Correct:
for (let i = 0; i < arr.length; i++) { }
```

**Mistake 2 — Modifying the array while iterating:**
```javascript
const nodes = ["a", "b", "c"];
for (let i = 0; i < nodes.length; i++) {
  nodes.push("d");  // array grows every iteration — infinite loop
}
```

**Mistake 3 — Using for...in on arrays:**
```javascript
const arr = [1, 2, 3];
for (const key in arr) {
  console.log(key);   // "0", "1", "2" — string indices, not values
  console.log(arr[key]);  // 1, 2, 3 — works but wrong tool
}
// Use for...of instead
```

**Mistake 4 — Forgetting that i++ runs AFTER the body:**
```javascript
for (let i = 0; i < 3; i++) {
  console.log(i);  // logs 0, 1, 2 — not 1, 2, 3
}
// If you want 1, 2, 3: log i + 1 or start with i = 1 and use i <= 3
```
