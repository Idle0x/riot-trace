## The Most Used Array Method

`.map()` is the most frequently used array method in JavaScript and in React. Every list you render, every dataset you transform, every API response you reshape — all of it uses `.map()`.

The core concept is simple: transform every element in an array into something new, and collect the results in a new array:

```javascript
const scores = [95, 47, 88, 71];
const doubled = scores.map(score => score * 2);
// [190, 94, 176, 142]
```

One array goes in. A new array comes out. The original is unchanged. The output array is always the same length as the input.

---

## How map Works Internally

You can think of `.map()` as a `for` loop that collects return values:

```javascript
// What map does internally (conceptual):
function map(array, transform) {
  const result = [];
  for (const item of array) {
    result.push(transform(item));  // collect each return value
  }
  return result;
}
```

This explains the most common mistake: forgetting `return` in a full-body callback. If the callback does not return anything, `undefined` is pushed into the result array:

```javascript
const scores = [95, 47, 88];

// Concise arrow — implicit return (no braces):
scores.map(score => score * 2)  // [190, 94, 176] ✓

// Full body — MUST use explicit return:
scores.map(score => {
  score * 2;         // no return — pushes undefined
})
// [undefined, undefined, undefined] ✗

scores.map(score => {
  return score * 2;  // explicit return
})
// [190, 94, 176] ✓
```

**Rule:** When you use `{}` braces in a map callback, you need `return`. When you omit braces (concise body), `return` is implicit.

## Transforming Objects — The Spread Pattern

The most powerful use of `.map()` is transforming arrays of objects. The spread-and-add pattern creates new objects with additional computed properties:

```javascript
const nodes = [
  { id: "7A5r", health: 95 },
  { id: "8BJT", health: 47 },
  { id: "Xk91", health: 91 }
];

const enriched = nodes.map(node => ({
  ...node,                                          // spread original
  status: node.health > 80 ? "healthy" : "at risk", // add computed property
  rank: null                                         // add placeholder property
}));
// [
//   { id: "7A5r", health: 95, status: "healthy", rank: null },
//   { id: "8BJT", health: 47, status: "at risk", rank: null },
//   { id: "Xk91", health: 91, status: "healthy", rank: null }
// ]
```

**Note the `( )` around `{ }`** in the arrow function: `node => ({ ...node })`. This is required. Without the parentheses, JavaScript cannot tell if `{` is the start of a function body or the start of an object literal. The parentheses make it unambiguous: this is an object.

```javascript
// Wrong — JavaScript thinks { is a function body:
nodes.map(node => { ...node, status: "healthy" })  // SyntaxError

// Correct — parens tell JS it's an object literal:
nodes.map(node => ({ ...node, status: "healthy" }))
```

## Extracting a Single Property

A simple but extremely common use of `.map()`: pull one property from each object into a flat array:

```javascript
const nodes = [
  { id: "7A5r", health: 95 },
  { id: "8BJT", health: 47 },
  { id: "Xk91", health: 91 }
];

const ids = nodes.map(node => node.id);
// ["7A5r", "8BJT", "Xk91"]

const healths = nodes.map(node => node.health);
// [95, 47, 91]
```

## map Always Returns the Same Length

This is a guarantee, not just a behavior: `.map()` always returns an array with exactly the same number of elements as the input:

```javascript
[1, 2, 3].map(n => n * 2)        // [2, 4, 6] — 3 in, 3 out
[1, 2, 3].map(n => "x")          // ["x", "x", "x"] — 3 in, 3 out
[1, 2, 3].map(n => undefined)    // [undefined, undefined, undefined] — 3 in, 3 out
```

If you want to produce fewer elements than the input, use `.filter()` (next lesson). If you want to collapse everything into a single value, use `.reduce()` (Lesson 18).

## When NOT to Use map

`.map()` is for transformation — producing a new array. Do not use it when:

**You only want side effects** (logging, network calls, DOM manipulation):
```javascript
// Wrong — using map for side effects:
nodes.map(node => console.log(node.id));  // returns [undefined, undefined, ...]
// Right — use forEach:
nodes.forEach(node => console.log(node.id));
```

**You want to filter elements out:**
```javascript
// Wrong — map always returns the same length:
const healthy = nodes.map(node => {
  if (node.health > 80) return node;
  // else returns undefined — healthy has undefined holes
});
// Right — use .filter():
const healthy = nodes.filter(node => node.health > 80);
```

## The Index and Array Parameters

Like `forEach`, `map`'s callback receives `(element, index, array)`:

```javascript
const nodes = ["7A5r", "8BJT", "Xk91"];

const ranked = nodes.map((id, index) => ({
  rank: index + 1,
  id: id
}));
// [{ rank: 1, id: "7A5r" }, { rank: 2, id: "8BJT" }, { rank: 3, id: "Xk91" }]
```

## Chaining map With Other Methods

Because `.map()` returns a new array, you can call another method on the result immediately:

```javascript
const nodes = [
  { id: "7A5r", health: 95, network: "mainnet" },
  { id: "8BJT", health: 47, network: "devnet"  },
  { id: "Xk91", health: 91, network: "mainnet" }
];

// Get IDs of mainnet nodes with health above 80:
const result = nodes
  .filter(node => node.network === "mainnet")
  .filter(node => node.health > 80)
  .map(node => node.id);
// ["7A5r", "Xk91"]
```

You will build full transformation pipelines like this in Lesson 24.

## Common Mistakes

**Mistake 1 — Missing return in full-body callback:**
```javascript
scores.map(score => {
  score * 2;    // no return → all elements become undefined
});
```

**Mistake 2 — Returning an object without parens in concise body:**
```javascript
nodes.map(node => { id: node.id })   // SyntaxError
nodes.map(node => ({ id: node.id })) // Valid
```

**Mistake 3 — Using map for side effects:**
```javascript
nodes.map(node => console.log(node.id));  // forEach is the right tool
```

**Mistake 4 — Expecting map to filter:**
```javascript
const healthy = nodes.map(n => n.health > 80 ? n : null);
// [node, null, node] — nulls pollute the result
// Use .filter() instead
```

**Mistake 5 — Mutating the original inside map:**
```javascript
nodes.map(node => {
  node.status = "processed";  // mutates the original — side effect
  return node;
});
// Use spread instead: return { ...node, status: "processed" };
```
