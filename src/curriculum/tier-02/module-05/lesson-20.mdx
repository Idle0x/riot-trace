This is arguably the most used array method in React. Every time you render a list of items to the screen (like a list of transactions in *riot-sop*), you are using `.map()`.

### Transformation, Not Just Iteration
`.forEach()` just runs code. It doesn't give you anything back. 
`.map()` is different. It transforms every element and **returns a brand new array** of the exact same length.

```javascript
const prices = [10, 20, 30];

// We catch the new array in a variable called "doubled"
const doubled = prices.map((price) => {
  return price * 2; 
});

console.log(doubled); // Logs: [20, 40, 60]
```

### Extracting Data
In system architecture, you often get massive objects from an API, but you only need one specific piece of data. `.map()` is perfect for plucking data out.

```javascript
const users = [
  { id: 1, name: "Alice", role: "admin" },
  { id: 2, name: "Bob", role: "user" }
];

// Extract just the names
const namesOnly = users.map((user) => {
  return user.name;
});

console.log(namesOnly); // Logs: ["Alice", "Bob"]
```

### The Golden Rule
When you use `.map()`, you **MUST** use the `return` keyword inside the block. Whatever you return is what gets placed into the new array. If you forget to return, you will get an array full of `undefined`.
