## The Test

Module 05 introduced the five iteration tools: `for` loops for manual control, `forEach` for side effects, `map` for transformation, `filter` for selection, `reduce` for aggregation, and `sort` for ordering.

This boss fight uses all of them in a single data pipeline — the same kind of pipeline that runs in every dashboard, every analytics system, and every data-heavy application.

---

## What This Tests

**Requirement 1** tests `forEach` for accumulation. You track a counter outside the loop and increment it for each node.

**Requirement 2** tests `filter` with combined conditions using `&&`.

**Requirement 3** tests `reduce` for a mathematical calculation followed by a conditional classification. The reduce produces the sum, you compute the average yourself, then use `if/else if` to get the label.

**Requirement 4** tests `sort` with a comparator and the rule that you must copy before sorting. Then `map` to extract IDs. Then access index 0.

**Requirement 5** tests `reduce` to an object — the grouping pattern. The initial value is `{ mainnet: 0, devnet: 0 }`. Each iteration adds the node's storage to `acc[node.network]`.

**Requirement 6** tests a full chain: sort a copy by health descending, `slice(0, 3)` for the top three, `map` to IDs, `join(",")` to produce the string with no spaces.

---

## Approach

Build and verify each requirement independently before combining. Do not attempt to write all six at once.

The most likely errors:
- Forgetting `return acc` in reduce callbacks
- Sorting without copying (`[...nodes]`)
- Wrong sort direction (`a - b` vs `b - a`)
- `join(",")` not `join(", ")` — no space in the expected output
