## Selective Inclusion

`.map()` transforms every element. `.filter()` decides which elements survive.

Where `.map()` always returns an array the same length as the input, `.filter()` returns an array that is equal in length or shorter — only the elements that pass the test are included:

```javascript
const scores = [95, 47, 88, 71, 55, 91];
const passing = scores.filter(score => score > 70);
// [95, 88, 71, 91]
// 47 and 55 did not pass the test — they are gone
```

Every filter button in your xandeum-pulse dashboard is this operation. The "mainnet" button calls something equivalent to `nodes.filter(n => n.network === "mainnet")`. The health threshold slider calls `nodes.filter(n => n.health >= threshold)`. Understanding `.filter()` means understanding exactly how filtering works in any application.

---

## How filter Works

The callback receives each element and must return a truthy or falsy value:

- Return truthy → element is **included** in the result
- Return falsy → element is **excluded** from the result

```javascript
const nodes = [
  { id: "7A5r", health: 95, online: true  },
  { id: "8BJT", health: 47, online: false },
  { id: "Xk91", health: 91, online: true  }
];

const online = nodes.filter(node => node.online);
// node.online is true → included
// node.online is false → excluded
// node.online is true → included
// Result: [{ id: "7A5r"... }, { id: "Xk91"... }]
```

Any truthy value keeps the element. Any falsy value removes it. This means you can filter by property existence, non-zero values, non-empty strings — anything JavaScript treats as truthy:

```javascript
const mixed = [0, 1, "", "hello", null, "world", undefined];
const truthy = mixed.filter(Boolean);
// [1, "hello", "world"] — only the truthy values survive
// Boolean is a function: Boolean(0) → false, Boolean(1) → true
```

## Filtering With Multiple Conditions

Use `&&` and `||` inside the callback to combine conditions:

```javascript
const nodes = [
  { id: "7A5r", health: 95, network: "mainnet", version: "1.2.1" },
  { id: "8BJT", health: 47, network: "mainnet", version: "1.2.0" },
  { id: "8PjP", health: 78, network: "devnet",  version: "1.2.1" },
  { id: "Xk91", health: 91, network: "devnet",  version: "1.2.1" }
];

// Both conditions must be true (&&):
const healthyMainnet = nodes.filter(
  node => node.network === "mainnet" && node.health > 70
);
// [{ id: "7A5r"... }] — only 7A5r is mainnet AND healthy

// Either condition is true (||):
const devnetOrOutdated = nodes.filter(
  node => node.network === "devnet" || node.version !== "1.2.1"
);
// [{ id: "8BJT"... }, { id: "8PjP"... }, { id: "Xk91"... }]
```

## filter Returns a New Array — Original Unchanged

Like `.map()`, `.filter()` is non-destructive:

```javascript
const nodes = ["node-A", "node-B", "node-C"];
const filtered = nodes.filter(n => n !== "node-B");

console.log(filtered); // ["node-A", "node-C"]
console.log(nodes);    // ["node-A", "node-B", "node-C"] — unchanged
```

The original array is never modified. `.filter()` reads it and creates a new one.

## Chaining filter With map

The pattern appears constantly in real codebases: filter to the relevant subset, then map to transform:

```javascript
const nodes = [
  { id: "7A5r", health: 95, network: "mainnet" },
  { id: "8BJT", health: 47, network: "mainnet" },
  { id: "8PjP", health: 78, network: "devnet"  },
  { id: "Xk91", health: 91, network: "devnet"  }
];

// Get IDs of healthy mainnet nodes:
const healthyMainnetIds = nodes
  .filter(node => node.network === "mainnet")  // 2 nodes remain
  .filter(node => node.health > 70)            // 1 node remains
  .map(node => node.id);                       // extract IDs
// ["7A5r"]
```

Each method in the chain receives the result of the previous method. Read it top-to-bottom as a pipeline: "from all nodes, keep only mainnet nodes, then from those keep only healthy ones, then extract their IDs."

## filter With Index

The callback also receives `(element, index, array)`:

```javascript
const nodes = ["a", "b", "c", "d", "e"];

// Keep every other item (even indices):
const evens = nodes.filter((item, index) => index % 2 === 0);
// ["a", "c", "e"]

// Remove the first item:
const withoutFirst = nodes.filter((item, index) => index !== 0);
// ["b", "c", "d", "e"]

// Keep only the first 3:
const firstThree = nodes.filter((item, index) => index < 3);
// ["a", "b", "c"] — but .slice(0, 3) is cleaner for this
```

## Removing Duplicates With filter

```javascript
const networks = ["mainnet", "devnet", "mainnet", "testnet", "devnet"];

const unique = networks.filter(
  (network, index, array) => array.indexOf(network) === index
);
// ["mainnet", "devnet", "testnet"]
// indexOf returns the FIRST occurrence index
// If this is not the first occurrence, indexOf < index → filtered out

// Modern alternative using Set:
const unique2 = [...new Set(networks)];
// Same result, cleaner code
```

## Common Mistakes

**Mistake 1 — Using map to filter (producing nulls):**
```javascript
const healthy = nodes.map(node => node.health > 80 ? node : null);
// [node, null, node] — nulls pollute the array
// Fix: use .filter()
```

**Mistake 2 — Missing return in full-body callback:**
```javascript
const filtered = nodes.filter(node => {
  node.health > 80;  // no return — always returns undefined (falsy) — filters everything out
});
// filtered: [] — empty!
```

**Mistake 3 — Mutating inside filter:**
```javascript
nodes.filter(node => {
  node.status = "processed";  // side effect — avoid this
  return node.health > 80;
});
```

**Mistake 4 — Expecting filter to transform:**
```javascript
const labels = nodes.filter(node => `${node.id}: ${node.health}%`);
// labels contains the original node objects, not the strings
// Non-empty strings are truthy — so this actually keeps everything
// Use .map() to transform
```
