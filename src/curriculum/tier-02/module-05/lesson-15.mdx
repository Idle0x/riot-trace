## Iteration With a Function

In Lesson 14 you used `for` and `for...of` to iterate manually. Both work. Both are explicit about their mechanics. But modern JavaScript codebases use a different approach for iteration: passing a function to the array that runs it for each element.

`forEach` is the first and most direct of these. It does exactly what `for...of` does — visits every element — but instead of a loop body you write a callback function:

```javascript
// for...of version:
for (const node of nodes) {
  console.log(node.id);
}

// forEach version:
nodes.forEach(node => {
  console.log(node.id);
});
```

Same result. The difference is style, expressiveness, and a set of specific capabilities that make `forEach` (and the methods that follow it) more powerful in certain contexts.

---

## The forEach Callback — Full Signature

The function you pass to `forEach` receives three arguments. You can use as many or as few as you need:

```javascript
array.forEach((element, index, array) => {
  // element — the current item
  // index   — its position (0, 1, 2...)
  // array   — the full original array
});
```

Most of the time you only need `element`:
```javascript
nodes.forEach(node => {
  console.log(node.health);
});
```

When you need the position, use `index`:
```javascript
nodes.forEach((node, index) => {
  console.log(`${index + 1}. ${node.id}`);
});
```

When you need to know the total count inside the callback, use `array`:
```javascript
nodes.forEach((node, index, arr) => {
  console.log(`Node ${index + 1} of ${arr.length}: ${node.id}`);
});
```

---

## What forEach Returns — And Why It Matters

`forEach` **always returns `undefined`**. Always. No exceptions.

```javascript
const result = nodes.forEach(node => node.id);
console.log(result);  // undefined — not an array of ids
```

This is the most important thing to understand about `forEach`: **it is for side effects only**. A side effect is anything that does something to the outside world — logging, modifying an external variable, pushing to an array defined outside the loop, making a network request.

```javascript
// forEach is the right tool when you want to DO something with each item:
nodes.forEach(node => console.log(node.id));         // side effect: logging
nodes.forEach(node => externalArray.push(node.id));  // side effect: array mutation
nodes.forEach(node => sendToServer(node));            // side effect: network call

// forEach is the WRONG tool when you want a new transformed array:
const ids = nodes.forEach(node => node.id);  // undefined — use .map() instead
```

The next lesson covers `.map()`, which returns a new array. If you want a transformed result, use `.map()`. If you want to do something with each item without caring about a result, use `forEach`.

---

## return Inside forEach Does Nothing

A common mistake: writing `return` inside a `forEach` callback expecting it to affect something:

```javascript
let total = 0;

nodes.forEach(node => {
  return total + node.health;  // return does nothing here
  // total is never updated
});

console.log(total);  // 0 — unchanged
```

`return` inside a `forEach` callback just exits that specific callback invocation. It does not break the loop (you cannot break `forEach` — more on that shortly). It does not accumulate a value. It does nothing useful.

The correct pattern for accumulating inside `forEach`:

```javascript
let total = 0;

nodes.forEach(node => {
  total += node.health;  // no return — mutate the outer variable directly
});

console.log(total);  // 233
```

---

## forEach Cannot break or continue

Unlike `for` and `for...of`, there is no way to stop a `forEach` loop early:

```javascript
// This does NOT stop the loop:
nodes.forEach(node => {
  if (node.health < 50) return;  // just skips this iteration
  console.log(node.id);
});
// Using return inside forEach is like continue in a for loop — skips the current item

// There is no equivalent of break in forEach
// To stop early, you must use a standard for loop or for...of with break
```

This is a deliberate design constraint. `forEach` is for processing all elements. If you need to stop early, use a different loop.

---

## forEach vs for...of — Choosing Between Them

Both iterate over every element. The choice comes down to two questions:

**Do you need to stop early?**
- Yes → use `for...of` with `break`
- No → either works

**Do you need the full array reference inside the callback?**
- Yes → `forEach` (the third parameter gives you the array)
- No → either works

In practice, functional code tends to prefer the array methods (`forEach`, `map`, `filter`) over explicit loops because they communicate intent more directly. But readability is the priority — if a `for` loop is clearer for a specific situation, use it.

## Common Mistakes

**Mistake 1 — Expecting forEach to return something:**
```javascript
const ids = nodes.forEach(node => node.id);
// ids is undefined — forEach always returns undefined
// Use .map() to get a new array
```

**Mistake 2 — Using return to accumulate:**
```javascript
let total = 0;
nodes.forEach(node => {
  return total + node.health;  // does nothing
});
// Fix: total += node.health;
```

**Mistake 3 — Trying to break forEach:**
```javascript
nodes.forEach(node => {
  if (condition) break;  // SyntaxError — break is not valid inside forEach
});
// Fix: use for...of with break, or use .find() / .some()
```

**Mistake 4 — Using forEach when you need a transformed array:**
```javascript
const labels = nodes.forEach(node => `${node.id}: ${node.health}%`);
// labels is undefined
// Fix: use .map() which collects and returns the results
```
