## Collapsing and Ordering

You now have three of the four core array transformation tools. `forEach` runs a function on each element. `map` transforms each element into something new. `filter` removes elements that don't qualify.

The fourth tool — `reduce` — does something different: it collapses an entire array into a single value. That value can be a number, a string, an object, or even another array.

`sort` is a different category: it reorders the elements of an array. Together, `reduce` and `sort` complete your data manipulation toolkit.

---

## reduce — The Swiss Army Knife

```javascript
array.reduce(callback, initialValue)
```

The callback receives two things:
- `acc` — the accumulator: the running result, starts as `initialValue`
- `current` — the current element being processed

Whatever the callback returns becomes the new `acc` for the next iteration:

```javascript
const scores = [95, 47, 88, 71];

const total = scores.reduce((acc, score) => {
  return acc + score;
}, 0);
// Iteration 1: acc=0,  score=95 → returns 95
// Iteration 2: acc=95, score=47 → returns 142
// Iteration 3: acc=142,score=88 → returns 230
// Iteration 4: acc=230,score=71 → returns 301
// Result: 301
```

The initial value `0` is the starting point for `acc`. If you omit it, `reduce` uses the first array element as the initial `acc` and starts processing from the second element — which works for simple sums but causes unexpected behavior in most other cases. **Always provide an initial value.**

---

## The Return Rule — Most Critical

The most common `reduce` bug: forgetting to return the accumulator:

```javascript
// WRONG — acc is updated but never returned:
const max = scores.reduce((acc, score) => {
  if (score > acc) acc = score;
  // no return!
}, 0);
// max is undefined — the callback returned nothing on the last iteration

// CORRECT — always return acc:
const max = scores.reduce((acc, score) => {
  if (score > acc) acc = score;
  return acc;
}, 0);

// Or with ternary:
const max = scores.reduce((acc, score) => score > acc ? score : acc, 0);
```

Every path through the reduce callback must return `acc`. If any iteration returns `undefined`, that becomes the new `acc`, and everything after that is wrong.

## reduce to a Number

```javascript
const transactions = [
  { amount: 50000, type: "inflow"  },
  { amount: 2000,  type: "outflow" },
  { amount: 15000, type: "outflow" },
  { amount: 8000,  type: "inflow"  }
];

// Sum all amounts:
const total = transactions.reduce((acc, tx) => acc + tx.amount, 0);
// 75000

// Net flow (add inflows, subtract outflows):
const net = transactions.reduce((acc, tx) => {
  return tx.type === "inflow"
    ? acc + tx.amount
    : acc - tx.amount;
}, 0);
// 41000

// Count items matching a condition:
const inflowCount = transactions.reduce(
  (acc, tx) => tx.type === "inflow" ? acc + 1 : acc,
  0
);
// 2
```

## reduce to an Object — Grouping and Aggregating

`reduce` becomes most powerful when the accumulator is an object:

```javascript
const nodes = [
  { id: "7A5r", network: "mainnet" },
  { id: "8BJT", network: "devnet"  },
  { id: "8PjP", network: "mainnet" },
  { id: "Xk91", network: "devnet"  }
];

// Group nodes by network:
const byNetwork = nodes.reduce((acc, node) => {
  const net = node.network;
  if (!acc[net]) acc[net] = [];    // create array if first in this group
  acc[net].push(node);             // add to the group
  return acc;                      // always return acc
}, {});
// {
//   mainnet: [{ id: "7A5r"... }, { id: "8PjP"... }],
//   devnet:  [{ id: "8BJT"... }, { id: "Xk91"... }]
// }

// Count by network:
const counts = nodes.reduce((acc, node) => {
  acc[node.network] = (acc[node.network] || 0) + 1;
  return acc;
}, {});
// { mainnet: 2, devnet: 2 }
```

## sort — Reordering Elements

`sort()` reorders the elements of an array **in place** (mutates the original) and returns the sorted array.

**Always sort a copy** unless you deliberately want to mutate the original:

```javascript
const scores = [95, 47, 88, 71];

// DANGER — sorts in place:
scores.sort();                       // mutates scores
const sorted = scores.sort();        // sorted and scores are the same array!

// SAFE — sort a copy:
const sorted = [...scores].sort();   // scores unchanged
```

---

## The Comparator Function

Without a comparator, `sort` converts everything to strings and sorts lexicographically (alphabetically). This breaks number sorting:

```javascript
[10, 1, 20, 2, 100].sort()
// [1, 10, 100, 2, 20] — alphabetical, NOT numeric!
```

The comparator function receives two elements `(a, b)` and must return:
- Negative number → `a` comes before `b`
- Zero → order unchanged
- Positive number → `b` comes before `a`

```javascript
// Numeric ascending (smallest first):
[...scores].sort((a, b) => a - b)
// a - b: if a < b → negative → a comes first (correct for ascending)

// Numeric descending (largest first):
[...scores].sort((a, b) => b - a)
// b - a: if b > a → positive → b comes first (correct for descending)

// Strings ascending:
["banana", "apple", "cherry"].sort((a, b) => a.localeCompare(b))

// Sorting objects by property:
nodes.sort((a, b) => b.health - a.health)  // descending by health
```

## Combining sort With Other Methods

```javascript
// Top 3 healthiest mainnet nodes:
const leaderboard = nodes
  .filter(node => node.network === "mainnet")
  .sort((a, b) => b.health - a.health)
  .slice(0, 3)
  .map((node, index) => ({
    rank: index + 1,
    id: node.id,
    health: node.health
  }));
```

Read it as a pipeline: filter → sort → take top 3 → format for display. This is precisely what your xandeum-pulse leaderboard page does.

## Common Mistakes

**Mistake 1 — Missing initial value in reduce:**
```javascript
[].reduce((acc, n) => acc + n)
// TypeError: Reduce of empty array with no initial value
// Always provide initial value: .reduce((acc, n) => acc + n, 0)
```

**Mistake 2 — Forgetting to return acc:**
```javascript
items.reduce((acc, item) => {
  acc.count++;
  // missing return acc!
}, { count: 0 });
```

**Mistake 3 — Sorting without copying:**
```javascript
const sorted = items.sort((a, b) => a - b);
// items itself is now sorted — use [...items].sort()
```

**Mistake 4 — Wrong sort direction:**
```javascript
// For descending, b - a (not a - b):
items.sort((a, b) => a - b)  // ascending — smallest first
items.sort((a, b) => b - a)  // descending — largest first
```

**Mistake 5 — Sorting numbers without a comparator:**
```javascript
[10, 1, 20, 2].sort()        // [1, 10, 2, 20] — string sort, wrong!
[10, 1, 20, 2].sort((a,b) => a - b)  // [1, 2, 10, 20] — numeric sort, correct
```
