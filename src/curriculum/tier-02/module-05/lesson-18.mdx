Before higher-order functions existed, all iteration was handled by manual loops. You still need to understand them to read legacy code and to understand what modern methods are doing underneath.



### The Standard `for` Loop
A standard loop has three parts separated by semicolons: `(Initialization; Condition; Update)`.

```javascript
for (let i = 0; i < 5; i++) {
  console.log(i);  // Logs 0, 1, 2, 3, 4
}
```
* **Initialization (`let i = 0`)**: Creates a counter variable. Runs once.
* **Condition (`i < 5`)**: Before every loop, the machine checks this. If true, it runs the code block. If false, the loop stops entirely.
* **Update (`i++`)**: After the block runs, the counter is increased by 1 (`i++` is shorthand for `i = i + 1`).

### The `for...of` Loop
When you have an array, the standard `for` loop is clunky. Modern JavaScript gives us `for...of`, which directly extracts each item from a collection.

```javascript
const nodes = ["nodeA", "nodeB", "nodeC"];

for (const node of nodes) {
  console.log(node); // Logs nodeA, then nodeB, then nodeC
}
```
*Read it as: "For every individual `node` inside the `nodes` array, do this..."*
