If you look at modern React code, you rarely see the word `function`. You see `=>` everywhere. This is the **Arrow Function**. It is just a shorter, cleaner way to write a function.



### The Syntax Translation

**1. The Old Way (Function Declaration)**
```javascript
function add(a, b) {
  return a + b;
}
```

**2. The New Way (Arrow Function)**
Notice how we store the function inside a `const` variable. We drop the word `function` entirely and put a fat arrow `=>` between the parameters and the code block.
```javascript
const add = (a, b) => {
  return a + b;
};
```

### The "Concise Body" (Implicit Return)
If your function only does **one thing** (a single expression), you can delete the `{ curly braces }` and delete the `return` keyword. It will automatically (implicitly) return whatever is on that line.

```javascript
// This does the exact same thing as the examples above!
const add = (a, b) => a + b;
```
*Note: If a function has no parameters, you must use empty parentheses `()`.*
```javascript
const ping = () => "pong";
```
