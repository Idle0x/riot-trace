Every React hook you use (`useState`, `useEffect`) depends on closures. If you don't understand closures, you will never truly understand React.



### What is a Closure?
When a function finishes running, any variables created inside it are destroyed to free up memory. 

**Unless**... that function *returns another function* that still needs those variables. 

A closure is created when an inner function "remembers" the variables from the scope where it was created, even after the outer function has finished running.

### The Mechanism
```javascript
function createMultiplier(multiplier) {
  
  // This inner function is returned to the outside world.
  // It "closes over" (remembers) the 'multiplier' variable.
  return function(number) {
    return number * multiplier;
  };
}

// 1. We run the outer function. It returns the inner function.
const double = createMultiplier(2);

// 2. createMultiplier has finished running. 
// BUT 'double' still remembers that multiplier is 2!
console.log(double(5)); // Logs: 10
```

### Why this matters
Closures allow you to build "factories" that generate customized functions, and they allow functions to hold onto private state (which is exactly how `useState` works under the hood).
