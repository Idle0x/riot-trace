A function is a reusable block of code that performs a specific task. Instead of writing the same calculation 50 times for 50 different nodes, you write it once and **call** it 50 times.



### Parameters vs Arguments
These two terms confuse every beginner. Let's lock them in:
* **Parameters:** The placeholders you define *inside* the function declaration.
* **Arguments:** The actual data you pass in when you *call* the function.

```javascript
// 'uptime' and 'storage' are PARAMETERS
function calculateHealth(uptime, storage) {
  const score = uptime + storage;
  return score;
}

// 8.5 and 250 are ARGUMENTS
const nodeHealth = calculateHealth(8.5, 250); 
```

### The Return Keyword
A function is like a factory machine. You feed raw materials in (Arguments), it processes them, and it spits a finished product out. 

If you do not use the `return` keyword, the machine finishes its work but drops the product on the floor. The function will silently evaluate to `undefined`. 

```javascript
function brokenAdd(a, b) {
  const sum = a + b;
  // Missing return!
}

const result = brokenAdd(5, 5); 
console.log(result); // Logs: undefined
```
