You have learned `.filter()` and `.map()`. Until now, you've used them one at a time. But in professional architecture, we string them together into a **Pipeline**.

### The Flow of Data
Because `.filter()` returns an array, and `.map()` returns an array, you can immediately call the next method without saving intermediate variables.

```javascript
const transactions = [
  { type: "inflow", amount: 500 },
  { type: "outflow", amount: 200 },
  { type: "inflow", amount: 1000 }
];

// Read top-to-bottom:
const formattedInflows = transactions
  .filter((tx) => tx.type === "inflow")      // 1. Keep only inflows
  .map((tx) => `Received: $${tx.amount}`);   // 2. Format them into strings

console.log(formattedInflows); 
// Logs: ["Received: $500", "Received: $1000"]
```

### Why Do This?
It makes your code read like a sentence. "Take the transactions, filter for inflows, map them to strings." It removes the need for clunky temporary variables.
