When you ask a database for information, it cannot give it to you instantly. Instead, it gives you a **Promise**. 

A Promise is exactly what it sounds like: an IOU. It is an object that says, *"I don't have the data right now, but I promise to let you know when I do."*



### The Three States of a Promise
1. **Pending:** The request is in flight. (Waiting for the database).
2. **Fulfilled (Resolved):** The data arrived successfully.
3. **Rejected:** Something went wrong (Network down, bad password).

### Handling the Promise (`.then` and `.catch`)
Because a Promise takes time, we cannot just assign it to a variable normally. 
```javascript
// WRONG: This just prints "Pending Promise", not the data!
const data = fetchNodeData(); 
console.log(data); 
```

To get the actual data, we chain `.then()` onto it. To handle errors, we chain `.catch()`.

```javascript
// RIGHT: 
fetchNodeData()
  .then((data) => {
    // This runs ONLY when the data arrives
    console.log("Success:", data); 
  })
  .catch((error) => {
    // This runs ONLY if it fails
    console.log("Failed:", error);
  });
```
