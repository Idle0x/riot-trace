Chaining `.then()` works, but if you have to make three or four database requests in a row, you end up with "Callback Hell" — deeply nested code that is incredibly hard to read.

Modern JavaScript introduced `async` and `await` to fix this.

### The Synchronous Illusion
`async/await` is just syntactic sugar over Promises. It allows you to write asynchronous code that *looks* like normal, top-to-bottom synchronous code.

```javascript
// You MUST put 'async' in front of the function definition
async function loadDashboard() {
  
  // The 'await' keyword pauses this specific function 
  // until the Promise resolves, then assigns the value!
  const user = await fetchUser();
  const nodes = await fetchNodes(user.id);
  
  console.log("Dashboard Loaded");
}
```

### The Rule of Await
You can **only** use the `await` keyword inside a function that has been labeled `async`. If you try to use `await` in the global scope or a normal function, the machine will crash.

### Try / Catch
Because we aren't using `.catch()` anymore, how do we handle errors? We wrap the code in a standard `try...catch` block.

```javascript
async function safeLoad() {
  try {
    const data = await riskyDatabaseCall();
    console.log(data);
  } catch (error) {
    console.log("Caught an error:", error);
  }
}
```
