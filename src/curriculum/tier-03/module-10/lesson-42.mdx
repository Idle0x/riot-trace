If you successfully completed the last lesson and opened your browser's real developer console, you would have seen a bright red warning: `Warning: Each child in a list should have a unique "key" prop.`

### Why React Needs Keys
React's entire job is to update the DOM as fast as possible. When you render a list of 100 nodes, and Node #42 gets deleted, React needs to know *which* specific `<div>` to remove from the glass. 

If you don't provide a unique identifier, React has to destroy the entire list and rebuild it from scratch, which destroys performance.



### Applying the Key
The `key` prop must be applied to the **outermost element** returned inside the `.map()` function. 
It must be a unique string or number (usually an ID from your database). **Do not use the array index** (e.g., 0, 1, 2) as a key unless the list will absolutely never change, reorder, or filter.

```jsx
const transactions = [
  { txHash: "0x123", amount: 50 },
  { txHash: "0x456", amount: 100 }
];

function Ledger() {
  return (
    <div>
      {transactions.map(tx => (
        // The key goes on the outermost wrapper!
        <div key={tx.txHash} className="ledger-row">
          <span>{tx.amount}</span>
        </div>
      ))}
    </div>
  );
}
```
