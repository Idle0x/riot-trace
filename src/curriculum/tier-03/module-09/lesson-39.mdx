You know how to use variables (`let x = 0`). But if you try to use a normal variable inside a React component and change it, **the screen will not update**. 

React does not watch normal variables. If you want the glass to change when data changes, you must use **State**.



### React.useState
State is simply "data that React watches." When state changes, React destroys the old HTML and instantly redraws the new HTML. We do this using the `useState` hook.

*(Note: Because we are in a sandbox without import statements, we access it via `React.useState`)*.

```jsx
function Counter() {
  // Destructuring! 
  // 'count' is the variable. 'setCount' is the specific function to change it.
  const [count, setCount] = React.useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicks: {count}
    </button>
  );
}
```

### The Rendering Cycle
Let's trace exactly what happens in the code above:
1. Component mounts. `count` is `0`. Button renders as `Clicks: 0`.
2. User clicks the button.
3. The `onClick` fires: `setCount(0 + 1)`.
4. React sees state was updated to `1`.
5. React completely **re-runs** the `Counter` function.
6. This time, `count` is `1`. Button re-renders as `Clicks: 1`.
