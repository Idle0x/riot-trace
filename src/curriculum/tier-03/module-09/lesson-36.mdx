When you write React, you type HTML directly inside JavaScript. This is called **JSX**. 

It looks like magic, but it is an illusion. The browser cannot read JSX. Before your code reaches the browser, a transpiler (like Babel or Next.js) reads your file and converts that HTML-looking syntax into pure JavaScript objects.



### What JSX Actually Is
When you write this:
```jsx
const button = <button>Click Me</button>;
```

Under the hood, React converts it into this:
```javascript
const button = React.createElement(
  "button",    // The tag name
  null,        // The props (classes, ids)
  "Click Me"   // The children (text inside)
);
```

### Components are just Functions
A React Component is simply a JavaScript function (exactly like you learned in Tier 2) that **returns JSX**. 

*Rule: Component names MUST start with a Capital Letter so React knows it's a custom component and not a standard HTML tag.*

```jsx
// This is a Component
function Header() {
  return <h1>Dashboard</h1>;
}
```
