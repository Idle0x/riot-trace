A button that does nothing is just a painted rectangle. To make the glass interactive, we have to listen for User Events.



### Listening to the DOM
In raw HTML, you might have seen `onclick="doSomething()"`. In React, the syntax changes slightly to camelCase: `onClick`.

Instead of passing a string of code, you pass an **Arrow Function** inside curly braces `{}`.

```jsx
function FirewallToggle() {
  
  // 1. Define the action
  const triggerFirewall = () => {
    console.log("Firewall Activated");
  };

  // 2. Attach the action to the element's onClick attribute
  return <button onClick={triggerFirewall}>Activate</button>;
}
```

### The Inline Arrow (Shortcut)
If your action is extremely simple (like a single `console.log`), you don't even need to define it on a separate line. You can write the arrow function directly inside the braces.

```jsx
function FirewallToggle() {
  // Written entirely inline
  return <button onClick={() => console.log("Activated")}>Activate</button>;
}
```

**CRITICAL RULE:** You must pass a *function definition*, not a *function execution*. 
* **RIGHT:** `onClick={() => console.log("Hi")}` (React will wait for the click).
* **WRONG:** `onClick={console.log("Hi")}` (React will execute this immediately when the page loads, before the user even clicks!).
