A component is just a function. But a function is useless if it can't accept data. In standard JavaScript, we pass data via *Arguments*. In React, we pass data via **Props** (short for Properties).



### The Props Object
When you use a component and pass it an attribute, React automatically bundles all attributes into a single object called `props` and hands it to your component.

```jsx
// 1. We define the component to accept 'props'
function Alert(props) {
  // We use curly braces { } to insert JavaScript variables inside JSX
  return <div>Alert: {props.message}</div>; 
}

// 2. We use the component and pass a 'message' prop
const ui = <Alert message="Database Offline" />;
```

### The Render Function
In your standard Next.js apps, Next handles the mounting for you. But in a raw environment, we have to tell React exactly what component to paint onto the screen.

I have injected a `render()` function into your Crucible. You hand it JSX, and it paints it to the white glass pane above the editor.

```jsx
function Button() {
  return <button>Click</button>;
}

// Paints the button to the glass
render(<Button />); 
```
