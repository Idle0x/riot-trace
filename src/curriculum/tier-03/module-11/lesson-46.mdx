An empty array `[]` tells the effect to run exactly once. But what if you want an effect to run *only* when a specific piece of data changes?



### Tracking Specific State
You can place variables inside the dependency array. React will watch them. If the component re-renders, React checks the array. If the variable is identical to the last render, it skips the effect. If the variable changed, it runs the effect.

```jsx
function Profile() {
  const [userId, setUserId] = React.useState(1);
  const [clicks, setClicks] = React.useState(0);

  // This will ONLY run when 'userId' changes.
  // It completely ignores 'clicks'.
  React.useEffect(() => {
    console.log("Fetching data for user: " + userId);
  }, [userId]); 

  // ... JSX ...
}
```

### The Three Rules of Dependencies
1. **No Array:** Runs on EVERY render (Extremely dangerous).
2. **Empty Array `[]`:** Runs exactly ONCE on mount (Great for initial setup).
3. **Filled Array `[data]`:** Runs on mount, AND whenever `data` changes.
