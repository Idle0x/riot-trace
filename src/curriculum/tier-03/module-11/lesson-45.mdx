A React component is just a factory machine that spits out JSX. It should never do "outside" work (like fetching data) while it is building the JSX. 

If you want the component to do outside work, you must use the `useEffect` escape hatch.



### The Anatomy of an Effect
`useEffect` takes two arguments:
1. **The Setup Function:** The code you want to run (the side effect).
2. **The Dependency Array:** The rules for *when* the effect should run.

```jsx
function App() {
  
  React.useEffect(() => {
    // 1. THIS RUNS AFTER THE JSX IS RENDERED
    console.log("Component has mounted to the screen!");
  }, []); // 2. EMPTY ARRAY = RUN EXACTLY ONCE
  
  // THIS RUNS FIRST
  return <div>Welcome</div>; 
}
```

### The Infinite Loop Danger
If you forget the empty array `[]` at the end of the `useEffect`, React will run your effect **every single time** the component re-renders. If your effect updates state, updating state causes a re-render... which runs the effect... which updates state. You just killed the browser.
