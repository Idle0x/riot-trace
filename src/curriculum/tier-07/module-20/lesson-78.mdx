When you build a platform like a sovereign financial operating system, you are actually building two completely separate applications that talk to each other.



### 1. The Client (Frontend)
This is the React code running in the user's browser. It is responsible for painting the glass (DOM), managing temporary State, and listening for User Events. **The Client cannot trust the user.** A user can open the developer tools and modify your React code. You never put secret API keys or raw database credentials in the Client.

### 2. The Server (Backend)
This is the Node.js/Python code running on a secure machine. It holds the database (`sqlite`, `postgres`), handles authentication, and processes heavy algorithms. **The Server is the source of truth.**

### The Boundary
The Client and Server are separated by the internet. They can only communicate via HTTP requests (`fetch`). Because of physical distance, this communication always takes time. 

In System Design, you must explicitly design for this delay. If you assume the database will reply instantly, your UI will feel broken and unresponsive to the user.
