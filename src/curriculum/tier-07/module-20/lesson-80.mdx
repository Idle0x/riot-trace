When the browser first loads your HTML, it is empty. It has no data. The process of fetching data from the Server and injecting it into the Client's React components is called **Hydration**.

### The Hydration Pipeline
You combine `useEffect` (for the side effect of fetching data) and your FSM (for managing the UI during the delay).

1. Component mounts (Status: `idle`).
2. `useEffect` fires immediately. It sets Status to `loading`.
3. The glass repaints to show a spinning wheel.
4. The `fetch` request goes out to the database.
5. The data returns. You save it into state (`setData`).
6. You set Status to `success`.
7. The glass repaints, mapping the new data to the screen.

If the network fails at step 5, you catch the error and set Status to `error`. 

### Your Assignment
You are hydrating the `NetworkView` component. 
Inside the `useEffect`, write the exact pipeline: set the status to `'loading'`, start a `setTimeout` for 500ms, and inside that timeout, inject the `mockDB` into your `data` state and flip the status to `'success'`.
