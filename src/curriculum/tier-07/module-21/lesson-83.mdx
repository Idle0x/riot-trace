### STAGE 2: MIDDLEWARE LOGIC
The data has been pulled from the database. It is now sitting in the backend server's RAM. Before we send it over the internet to the React frontend, we must process it. 

### EXECUTION REQUIREMENTS
You are receiving an array of 100,000 node objects. 
`{ id: '1A', latency: 45 }`

You must write an **$O(N)$ Time Complexity** algorithm to find the node with the lowest `latency`.
* You cannot use `array.sort()`. Sorting an array takes $O(N \log N)$ time. For 100,000 items, sorting is too slow and will trigger a `TIME LIMIT EXCEEDED` failure.
* **The Strategy:** Create a variable to track the `lowestLatency` seen so far, and another for the `bestNodeId`. Loop through the array exactly **once**. If the current node's latency is lower than your tracker, update the tracker.

Return the `id` of the winning node.
