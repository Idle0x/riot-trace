### STAGE 3: THE GLASS
This is the absolute final milestone of the curriculum. The database is secure. The routing algorithm is $O(N)$. Now, you must paint the glass.

You are building the full React UI for the Xandeum Pulse Dashboard. It must be resilient, state-driven, and perfectly mapped.

### EXECUTION REQUIREMENTS
1.  Initialize a Finite State Machine (`status`) starting at `'idle'`.
2.  Initialize your data state (`nodes`) as an empty array `[]`.
3.  Complete the `syncNetwork` function. It must set the status to `'loading'`, wait 400ms using `setTimeout`, and then hydrate the `nodes` state with the `mockNetworkData` while flipping the status to `'success'`.
4.  Write a `useEffect` hook with an empty dependency array `[]` that calls `syncNetwork()` so the dashboard automatically fetches data the second it opens.
5.  Write your conditional UI returns.
    * If `'loading'`, return a `<div>` that says `"Syncing Pulse..."`.
    * If `'success'`, return a `<div>`. Inside it, `.map()` over your `nodes` and render their names. **Crucially**, also include a `<button>` with an `onClick` that triggers `syncNetwork()` again, so the user can manually refresh the dashboard.

*Validation Protocol:* The engine will verify your effect dependencies, check for the loading state implementation, scan the DOM for the properly mapped node names, and verify the physical existence of your interactive Refresh button.

Execute.
