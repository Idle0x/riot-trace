### STAGE 1: DATA RETRIEVAL
Every full-stack application begins at the database. Before the UI can render anything, the backend must extract the correct data from the ledger efficiently.

I have seeded the WebAssembly engine with two tables for the Xandeum Pulse network:
1. `nodes` (`id`, `name`, `status`)
2. `metrics` (`id`, `health`, `node_id`)

### EXECUTION REQUIREMENTS
Write a single, highly optimized SQL query to find the best node for the next transaction block.
1. `SELECT` all columns.
2. `JOIN` the `nodes` and `metrics` tables on their foreign key relationship.
3. Filter the results so you only look at nodes `WHERE` the `status` is `'online'`.
4. `ORDER BY` the `health` metric in descending order (highest health first).
5. `LIMIT` the output to exactly `1` row.

Execute the query. Prove your backend architecture is secure.
